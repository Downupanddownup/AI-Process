# 03-执行分支与旧逻辑清理

## 目标

将 `复执行` 的复制逻辑从“复选框二元分支”重构为“策略下拉框驱动的多分支逻辑”，并清理旧的无用代码和旧模板引用。

## 范围

1. 改造 `CopyExecutePrompt()`。
2. 建立“策略 -> 模板 -> 反馈文案”的调用链。
3. 清理 `ExecuteStepCheckbox` 及其相关旧逻辑。
4. 处理旧执行模板文件引用的移除。
5. 做本次改动的回归验证。

## 具体实施内容

### 1. 改造 `CopyExecutePrompt()`

目标逻辑：

1. 检查当前目录下是否存在 `实施文档.md`。
2. 读取执行策略下拉框当前选项。
3. 通过映射确定模板文件。
4. 读取模板内容。
5. 替换 `{{filePath}}`。
6. 写入剪贴板。
7. 根据策略显示对应反馈消息。

## 2. 保持已有行为约束

必须保持：

1. 未找到 `实施文档.md` 时仍报错并停止。
2. `禁改正式` 不受影响。
3. `实` 复选框逻辑不受影响。

## 3. 清理旧逻辑

必须删除或替换：

1. `ExecuteStepCheckbox` 全局变量。
2. 旧复选框创建代码。
3. 旧复选框启用/禁用代码。
4. `CopyExecutePrompt()` 中基于勾选状态切换 `execute_prompt.txt` / `execute_step_prompt.txt` 的分支。
5. `EnsureDefaultFiles()` 中旧执行双模板的直接依赖。

目标是不留下“已废弃但还能跑”的旧代码路径。

## 4. 旧模板文件处理

根据实施时具体策略决定：

1. 如果确认不会再被任何逻辑引用，可删除旧执行模板文件。
2. 如果暂时保留文件本身，也必须确保主脚本不再引用它们。

最终目标仍是“不留无用代码和无用依赖”。

## 5. 回归验证

重点验证：

1. `直接执行` 复制结果正确。
2. `AI判断` 复制结果正确。
3. `步骤文档` 复制结果正确。
4. `步骤目录` 复制结果正确。
5. `{{filePath}}` 替换正确。
6. 其他按钮行为未回归。

## 产出结果

完成本步骤后，本次需求应整体落地完成，旧结构被新结构完全替换。

## 验收点

1. `复执行` 已完全受下拉框策略驱动。
2. 4 种执行策略全部可用。
3. 旧 `步` 相关逻辑已清理干净。
4. 执行类模板引用已经全部切换到新目录结构。
5. 关键功能回归验证通过。
