# 实施文档：AI Process "打开 Markdown" 功能

## 一、背景与目标

当前 AI Process 已支持用 IDEA 打开 `需求.txt` 和 `对vN的回复.txt` 等文本文件。为了形成完整闭环，需要新增一个开关：当 AI 在回复过程中自行生成 Markdown 文件（`vN.md` 或 `实施文档.md`）后，能够自动用 IDEA 打开该文件。

## 二、需求边界

### 2.1 功能范围

- 在主面板新增一个复选框，标签为**"MD开"**（注意带空格）。
- 该开关与现有"IDEA开"开关为**平行关系**，互不影响。
- 开关**默认开启**。
- 仅影响以下 3 个提示词复制入口：
  1. **复需求**（`CopyRequirementPrompt`）
  2. **复回复**（`CopyReplyPrompt`）
  3. **复关系**（`CopyContextRelations`）
- 当开关开启时，在上述入口复制的提示词末尾追加一段指令，告诉 AI：如果生成了 Markdown 文件，生成完成后用 IDEA 打开它。
- 打开方式通过 PowerShell 脚本完成，脚本读取 `settings.ini` 中的 `IdeaCommand`，然后调用 IDEA 打开指定 Markdown 文件。
- **只支持 IDEA**，不支持其他编辑器。
- **只使用绝对路径**，不使用相对路径。
- 失败时输出错误提示。

### 2.2 关键原则

- **脚本路径**：由 AIProcess 在发送提示词前预填入，模型不需要自己寻找脚本位置。
- **Markdown 文件路径**：由模型自己提供。AIProcess 不在提示词中预填具体 Markdown 文件路径，因为只有模型知道自己实际创建了什么文件。

### 2.3 不在本次范围

- 不由 AIProcess 主动监听文件系统变化来打开 Markdown。
- 不修改现有"IDEA开"的行为。
- 不支持除 `vN.md` 和 `实施文档.md` 之外的其他 Markdown 文件类型（如步骤文档 `01-xxx.md`，本次不特殊处理，但脚本本身可打开任意 `.md`）。
- 不修改安装/卸载脚本（不需要加入 PATH）。

## 三、总体架构

```text
用户点击"复需求/复回复/复关系"
        ↓
AIProcess 拼接提示词
        ↓
如果"MD开"开关开启：
  追加打开 Markdown 的指令（含 PowerShell 脚本绝对路径）
        ↓
提示词复制到剪贴板，自动发送给 Agent
        ↓
AI Agent 生成 Markdown 文件后，将提示词中的 <Markdown文件绝对路径>
替换为实际文件路径，然后执行 PowerShell 命令
        ↓
OpenMarkdown.ps1 读取 settings.ini 中的 IdeaCommand
        ↓
用 IDEA 打开 Markdown 文件
```

## 四、详细设计

### 4.1 PowerShell 脚本：`app/OpenMarkdown.ps1`

**职责**：接收 Markdown 文件绝对路径，读取 `settings.ini` 中的 IDEA 路径，调用 IDEA 打开文件。

**参数**：

- `-FilePath`：要打开的 Markdown 文件绝对路径（必填）。

**行为**：

1. 根据脚本所在位置，自动定位 `app/config/settings.ini`。
2. 解析 INI，读取 `[Editor]` 段下的 `IdeaCommand`。
3. 校验：
   - `settings.ini` 存在。
   - `IdeaCommand` 已配置。
   - IDEA 可执行文件存在。
   - 目标 Markdown 文件存在。
4. 使用 `Start-Process` 调用 IDEA 并传入文件路径。
5. 调用完成后立即返回，不阻塞。

**脚本内容**：

```powershell
<#
.SYNOPSIS
    Opens a Markdown file in IntelliJ IDEA.

.DESCRIPTION
    Reads the IDEA executable path from AIProcess settings.ini and uses it
    to open the specified Markdown file.

.PARAMETER FilePath
    Absolute path to the Markdown file to open.
#>
[CmdletBinding()]
param(
    [Parameter(
        Mandatory = $true,
        Position = 0,
        HelpMessage = "Absolute path to the Markdown file"
    )]
    [ValidateNotNullOrEmpty()]
    [string]$FilePath
)

$ErrorActionPreference = "Stop"

# Resolve the path to settings.ini relative to this script's location.
# This script is expected to reside in the AIProcess app directory,
# alongside the config folder.
$scriptDirectory = Split-Path -Parent $MyInvocation.MyCommand.Path
$settingsPath = Join-Path $scriptDirectory "config\settings.ini"

function Read-IniValue {
    <#
    .SYNOPSIS
        Reads a single value from an INI file.
    #>
    param(
        [Parameter(Mandatory = $true)]
        [string]$Path,

        [Parameter(Mandatory = $true)]
        [string]$Section,

        [Parameter(Mandatory = $true)]
        [string]$Key
    )

    $currentSection = $null

    foreach ($line in Get-Content -Path $Path -Encoding UTF8) {
        $trimmedLine = $line.Trim()

        if ([string]::IsNullOrWhiteSpace($trimmedLine) -or
            $trimmedLine.StartsWith(";")) {
            continue
        }

        if ($trimmedLine -match "^\[(.+)\]$") {
            $currentSection = $matches[1]
            continue
        }

        if ($currentSection -eq $Section -and
            $trimmedLine -match "^(.+?)\s*=\s*(.*)$") {
            $currentKey = $matches[1].Trim()
            if ($currentKey -eq $Key) {
                return $matches[2].Trim()
            }
        }
    }

    return $null
}

function Assert-PathExists {
    <#
    .SYNOPSIS
        Verifies that a file or directory exists, otherwise exits with an error.
    #>
    param(
        [Parameter(Mandatory = $true)]
        [string]$Path,

        [Parameter(Mandatory = $true)]
        [string]$Description
    )

    if (-not (Test-Path -Path $Path)) {
        Write-Error "$Description not found: $Path"
        exit 1
    }
}

# Validate settings file exists.
Assert-PathExists -Path $settingsPath -Description "settings.ini"

# Read IDEA command from settings.
$ideaCommand = Read-IniValue -Path $settingsPath -Section "Editor" -Key "IdeaCommand"
if ([string]::IsNullOrWhiteSpace($ideaCommand)) {
    Write-Error "IdeaCommand is not configured in settings.ini [Editor] section."
    exit 1
}

# Validate IDEA executable and target Markdown file exist.
Assert-PathExists -Path $ideaCommand -Description "IDEA executable"
Assert-PathExists -Path $FilePath -Description "Markdown file"

# Open the Markdown file with IDEA.
Start-Process -FilePath $ideaCommand -ArgumentList "`"$FilePath`"" -NoNewWindow

Write-Host "Opened '$FilePath' in IDEA."
```

### 4.2 提示词模板：`app/templates/open_md_prompt.txt`

```text
补充指令：如果你在本轮回复中生成了 Markdown 文件，请在文件保存完成后，执行以下命令用 IDEA 打开它。注意：不要修改脚本路径，只需把 `<Markdown文件绝对路径>` 替换为你实际生成的 Markdown 文件绝对路径。

powershell -ExecutionPolicy Bypass -File "{{scriptPath}}" -FilePath "<Markdown文件绝对路径>"
```

**占位符说明**：

- `{{scriptPath}}`：由 AIProcess 替换为 `app/OpenMarkdown.ps1` 的绝对路径。
- `<Markdown文件绝对路径>`：保留给模型替换。模型知道自己生成了什么文件，因此由它填入实际路径。

#### 4.3.7 追加打开 Markdown 指令

新增函数：

```ahk
AppendOpenMdPromptIfNeeded(content) {
    global AppConfig, TemplateDir
    if (!AppConfig["OpenMdWithIdea"]) {
        return content
    }

    templatePath := TemplateDir "\open_md_prompt.txt"
    if (!FileExist(templatePath)) {
        return content
    }

    template := FileRead(templatePath, "UTF-8")
    template := StrReplace(template, "{{scriptPath}}", AppConfig["OpenMdScriptPath"])

    return RTrim(content, "`r`n") "`r`n`r`n" template
}
```

## 七、风险与注意事项

1. **Agent 执行权限**
   - 该方案依赖 AI Agent 能够执行 PowerShell 命令。
   - Kimi Code CLI、Claude Code、Codex CLI 均支持；Cloud Code 等云端环境可能受限。

2. **模型是否正确替换 Markdown 路径**
   - 提示词中已明确要求"把 `<Markdown文件绝对路径>` 替换为你实际生成的 Markdown 文件绝对路径"。
   - 模型知道自己创建了什么文件，因此由它提供路径是最准确的。

3. **路径中包含空格**
   - `IdeaCommand` 和 Markdown 文件路径可能包含空格，PowerShell 脚本中已用双引号包裹。

4. **脚本位置固定**
   - 提示词中嵌入的是 `app/OpenMarkdown.ps1` 的绝对路径。如果 AIProcess 被移动到新的目录，需要重新启动 AIProcess 以更新提示词中的路径。

5. **失败提示**
   - 脚本中通过 `Write-Error` 输出错误信息，Agent 执行失败时会看到错误输出。
