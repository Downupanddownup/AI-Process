## 四、AI 直接打开 IDEA 的实现方案

### 4.1 核心思路

在发送给 AI 的提示词中，追加一段明确指令：

> 如果你在回复中创建了 Markdown 文件，请在文件保存完成后，执行以下 Windows 命令用 IDEA 打开它（将 `{{mdFilePath}}` 替换为你实际创建的文件路径）：
> ```
> {{openMdCommand}}
> ```

其中：

- `{{mdFilePath}}`：AI 实际生成的 `.md` 文件路径。
- `{{openMdCommand}}`：AIProcess 根据 `settings.ini` 中的 `IdeaCommand` 预填充的命令，例如：
  ```text
  "C:\NotSystem\app\idea\IntelliJ IDEA 2023.3.8\bin\idea64.exe" "{{mdFilePath}}"
  ```

### 4.2 追加位置

只在以下 3 个入口的提示词中追加：

1. **复需求** (`CopyRequirementPrompt()`)
2. **复回复** (`CopyReplyPrompt()`)
3. **复关系** (`CopyContextRelations()`)

不在"复执行"中追加，因为执行阶段 AI 的角色是修改代码，不是生成沟通文档。

### 4.3 追加时机

当且仅当"打开MD"复选框被勾选时追加。与现有"禁改正式"追加逻辑类似，通过一个函数 `AppendOpenMdPromptIfNeeded(content)` 统一处理。

### 4.4 新增模板文件

建议新增模板文件 `app/templates/open_md_prompt.txt`，内容大致如下：

```text
补充指令：如果你在回复过程中创建了 Markdown 文件（如 vN.md 或 实施文档.md），请在文件保存完成后，用 IDEA 打开它。可执行命令如下（将 {{mdFilePath}} 替换为实际文件路径）：
{{openMdCommand}}
```

## 五、仍需确认的关键细节

### 5.1 AI 执行命令的权限与环境

你说"需要 AI Agent 能够直接打开 IDEA"，但不同 Agent 的执行能力不同：

| Agent | 是否能执行本地命令 | 说明 |
|---|---|---|
| **Codex CLI** | 通常可以 | 本地终端 Agent，可调用 PowerShell/CMD |
| **Claude Code** | 通常可以 | 本地终端 Agent，有 shell 工具 |
| **Cloud Code** | 取决于运行环境 | 云端/IDE 插件形式，权限可能受限 |

**问题**：你主要使用哪一种？如果混合使用，是否需要提供多种命令格式（CMD / PowerShell）让 AI 选择？

### 5.2 命令格式选择

Windows 下常见的打开命令格式有几种：

**CMD 风格（最通用）**：
```cmd
"C:\...\idea64.exe" "C:\...\file.md"
```

**PowerShell 风格**：
```powershell
Start-Process "C:\...\idea64.exe" -ArgumentList '"C:\...\file.md"'
```

**AHK 风格（如果 AI 能执行 AHK 代码）**：
```ahk
Run('"C:\...\idea64.exe" "C:\...\file.md"')
```

**建议**：提供 CMD 风格即可，因为 Codex / Claude Code 在 Windows 下通常都能执行。但路径中含有空格，需要确保双引号转义正确。

### 5.3 `{{mdFilePath}}` 如何确定

不同入口能预知的 Markdown 路径不同：

| 入口 | 可预知的 Markdown 路径 |
|---|---|
| 复需求 | 无法确定，只能给示例 |
| 复回复 | 通常是 `v(N+1).md`，即 `nextVersionFile` |
| 复关系 | 无法确定，只能给示例 |

**建议**：
- 对于"复回复"，可以直接给出 `v(N+1).md` 的完整路径。
- 对于"复需求"和"复关系"，给出示例路径（如 `C:\...\v1.md`），并说明"替换为实际生成的 Markdown 文件路径"。

### 5.4 是否需要检测命令执行结果

AI 执行命令后可能会失败（例如 IDEA 未安装、路径错误）。是否需要：

- a) AI 自己读取错误输出并报告？
- b) 不做处理，由用户自行发现？

你之前说"失败就提示个错误"，这里指的是 AI 提示错误，还是 AIProcess 弹窗提示？
