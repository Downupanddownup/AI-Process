# 05：测试验证

## 目标

验证"MD开"功能在 UI、提示词、脚本执行三个层面都按预期工作。

## 测试环境

- AIProcess 主程序：`app/AIProcess.ahk`
- PowerShell 脚本：`app/OpenMarkdown.ps1`
- 提示词模板：`app/templates/open_md_prompt.txt`
- IDEA 路径：从 `app/config/settings.ini` 的 `[Editor] IdeaCommand` 读取

## 测试用例

### 用例 1：UI 显示与默认状态

**步骤**：

1. 启动 AIProcess。
2. 观察主面板第 4 行（绑窗口 / 解绑 所在行）。

**预期结果**：

- "绑窗口"按钮左侧出现"MD开"复选框。
- 复选框默认处于勾选状态。
- 窗口高度与之前一致，未因新增控件而变高。

### 用例 2：配置持久化

**步骤**：

1. 取消勾选"MD开"。
2. 关闭 AIProcess。
3. 重新启动 AIProcess。

**预期结果**：

- "MD开"复选框保持未勾选状态。
- `app/config/settings.ini` 中 `[Editor] OpenMdWithIdea` 值为 `0`。

### 用例 3：复需求提示词

**步骤**：

1. 设置任意主题目录。
2. 确保"MD开"已勾选。
3. 点击"复需求"。
4. 将剪贴板内容粘贴到记事本。

**预期结果**：

- 提示词末尾包含：
  ```text
  powershell -ExecutionPolicy Bypass -File "C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1" -FilePath "<Markdown文件绝对路径>"
  ```
- 脚本路径为绝对路径。
- Markdown 路径位置为 `<Markdown文件绝对路径>` 占位符。

### 用例 4：复回复提示词

**步骤**：

1. 设置一个包含 `vN.md` 的目录。
2. 确保"MD开"已勾选。
3. 点击"复回复"。
4. 将剪贴板内容粘贴到记事本。

**预期结果**：

- 提示词末尾同样包含 PowerShell 打开命令。
- 脚本路径正确。
- Markdown 路径位置为占位符。

### 用例 5：复关系提示词

**步骤**：

1. 设置任意主题目录。
2. 确保"MD开"已勾选。
3. 点击"复关系"。
4. 将剪贴板内容粘贴到记事本。

**预期结果**：

- 提示词末尾同样包含 PowerShell 打开命令。
- 脚本路径正确。
- Markdown 路径位置为占位符。

### 用例 6：PowerShell 脚本执行

**步骤**：

1. 在主题目录下创建一个测试 Markdown 文件，例如 `测试.md`。
2. 从"复需求"提示词中复制 PowerShell 命令。
3. 手动将 `<Markdown文件绝对路径>` 替换为 `测试.md` 的绝对路径。
4. 在终端执行该命令。

**预期结果**：

- IDEA 启动或激活。
- `测试.md` 在 IDEA 中打开。

### 用例 7：关闭开关后不再追加

**步骤**：

1. 取消勾选"MD开"。
2. 点击"复需求"。
3. 将剪贴板内容粘贴到记事本。

**预期结果**：

- 提示词末尾不包含任何关于打开 Markdown 的 PowerShell 命令。

### 用例 8：脚本错误处理

**步骤**：

1. 临时修改 `settings.ini` 中的 `IdeaCommand` 为一个不存在的路径。
2. 执行 PowerShell 脚本打开任意 Markdown 文件。

**预期结果**：

- 脚本输出错误信息，提示 IDEA 可执行文件未找到。
- 不弹出无关窗口。

## 回归测试

- 确认现有"IDEA开"功能不受影响。
- 确认"禁改正式"、"自隐藏"等其他复选框功能正常。
- 确认"复执行"入口不追加打开 Markdown 指令。

## 验收标准

以上 8 个用例全部通过，即可认为本功能实施完成。
