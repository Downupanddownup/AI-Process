# V6：脚本放置位置与模型可访问性分析

## 一、问题的核心

你提出："最终脚本的位置，肯定要放在模型能够接触到的路径下。"

这里的"模型能够接触到"，我理解包含两层含义：

1. **文件系统层面可访问**：模型执行 Shell 命令时，能读取到该脚本文件。
2. **信息层面可获取**：模型在提示词中能明确知道脚本的完整调用路径或命令。

由于模型本身没有记忆上一个会话的绝对路径，**最可靠的方式是在每次提示词中直接给出脚本的完整绝对路径**。

## 二、候选方案分析

### 方案 A：放在 AIProcess 安装目录，提示词中嵌入绝对路径

**路径示例**：

```text
C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1
```

**模型调用方式**：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1" -FilePath "C:\...\file.md"
```

**优点**：

- 路径稳定，不依赖模型当前工作目录。
- 不需要修改系统 PATH。
- 与现有 `settings.ini` 中 `IdeaCommand` 使用绝对路径的方式一致。
- AIProcess 自身通过 `A_ScriptDir` 就能计算出脚本路径，自动填充到提示词中。

**缺点**：

- 如果 AIProcess 被移动到另一个目录，提示词中的路径会失效（但这种情况极少）。

### 方案 B：添加到系统 PATH，模型按名称调用

**安装时操作**：

将 `C:\NotSystem\ClaudeCodeTool\AIProcess\app` 添加到系统 PATH。

**模型调用方式**：

```powershell
powershell -ExecutionPolicy Bypass -File OpenMarkdown.ps1 -FilePath "C:\...\file.md"
```

**优点**：

- 提示词更短，不暴露完整安装路径。
- 模型调用时更自然。

**缺点**：

- 需要修改 install.ps1/install.bat，安装时写入 PATH。
- PATH 修改后可能需要重启终端才能生效，模型当前会话可能读不到。
- 如果 PATH 中有同名脚本，可能产生冲突。

### 方案 C：复制脚本到当前工作目录

**操作**：

每次复制提示词时，AIProcess 把 `OpenMarkdown.ps1` 拷贝到 `CurrentDir`。

**模型调用方式**：

```powershell
powershell -ExecutionPolicy Bypass -File ".\OpenMarkdown.ps1" -FilePath "file.md"
```

**优点**：

- 路径短，模型容易理解。

**缺点**：

- 模型当前工作目录不一定等于 `CurrentDir`，相对路径可能失效。
- 每个主题目录都多一个脚本文件，污染工作区。
- 复制文件有副作用，与"机械操作由 AIProcess 做"的初衷冲突。

### 方案 D：放在系统级固定目录

**路径示例**：

```text
C:\Program Files\AIProcess\OpenMarkdown.ps1
```

**优点**：

- 与具体项目位置解耦。
- 更符合传统软件安装规范。

**缺点**：

- 当前 AIProcess 是绿色/便携模式，安装到 `C:\NotSystem\ClaudeCodeTool\AIProcess`。
- 改成系统目录需要重构安装脚本。
- 并不比方案 A 更可靠。

## 三、我的推荐

**推荐方案 A：脚本放在 `app/OpenMarkdown.ps1`，提示词中嵌入绝对路径。**

理由：

1. **与现有设计一致**：`settings.ini` 中的 `IdeaCommand` 已经是绝对路径，说明项目本身接受绝对路径配置。
2. **零额外安装副作用**：不需要改 PATH、不需要复制文件、不需要系统目录权限。
3. **模型调用最可靠**：绝对路径避免了工作目录不确定的问题。
4. **AIProcess 自动填充**：用户完全不需要手动输入路径。

## 四、如何让模型"准确执行"

除了路径本身，提示词中的指令格式也会影响模型执行准确率。我建议在提示词中给出完整、不可改动的命令模板：

```text
如果你在本轮回复中生成了 Markdown 文件，请在文件保存完成后，执行以下命令用 IDEA 打开它。
注意：不要修改命令中的脚本路径，只需把 {{mdFilePath}} 替换为你实际生成的 Markdown 文件绝对路径。

powershell -ExecutionPolicy Bypass -File "C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1" -FilePath "{{mdFilePath}}"
```

这样模型只需要做一件事：替换 `{{mdFilePath}}` 为实际路径。其他部分原样复制执行即可。

## 五、对脚本本身的简化建议

为了让模型更容易执行，脚本调用方式应尽量简单。当前脚本只有一个 `-FilePath` 参数，已经很简单。但还可以进一步优化：

- **支持相对路径**：如果模型传入相对路径，脚本自动基于 `CurrentDir` 解析（但这需要脚本知道 `CurrentDir`，实现较复杂）。
- **保持绝对路径要求**：要求模型必须传绝对路径，简单明确。

我倾向后者：**要求模型传 Markdown 文件的绝对路径**，避免相对路径带来的不确定性。

## 六、最终方案草案

```text
脚本位置：C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1
调用命令：powershell -ExecutionPolicy Bypass -File "C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1" -FilePath "<Markdown文件绝对路径>"
提示词填充：由 AIProcess 自动替换 {{scriptPath}} 和 {{mdFilePath}}
```

3. **脚本调用时，是否允许模型使用相对路径？**（我建议不允许，统一用绝对路径）
