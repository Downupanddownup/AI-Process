# V7：对 实施文档.md 的修正与确认

## 一、你的反馈梳理

你针对 `实施文档.md` 提出了三点关键意见：

1. **复选框标签**：必须是"打开 MD"（带空格），不能用"打开 Markdown"全称，放不下。
2. **脚本路径填充方式**：脚本路径必须由 AIProcess 在发送前预填入提示词，不能让模型自己找。
3. **Markdown 文件路径**：必须由大模型自己提供，AIProcess 不应预填。因为只有模型知道自己创建了什么文件。

另外你强调：即使是小脚本，也要注意代码质量、可读性、可维护性，不能为了省事破坏原有逻辑。

## 二、我的理解与修正

### 2.1 复选框标签

确认改为：

```text
打开 MD
```

不是"打开MD"，也不是"打开 Markdown"。

### 2.2 脚本路径

我的原意确实是 AIProcess 预填脚本路径，但文档表述可能不够明确。修正后：

- `{{scriptPath}}` 由 AIProcess 根据 `A_ScriptDir` 自动替换为 `app/OpenMarkdown.ps1` 的绝对路径。
- 模型只需要原样复制执行，不需要自己找脚本位置。

### 2.3 Markdown 文件路径

这是需要重点修正的地方。原设计让 AIProcess 预填了 `v1.md`、`v(N+1).md`、`实施文档.md`，这是错误的。修正后：

- Markdown 文件路径不由 AIProcess 预填。
- 提示词中使用一个明确的占位符 `<Markdown文件绝对路径>`，由模型根据实际生成的文件自行替换。
- 脚本只负责接收路径并执行打开，不猜测文件位置。

### 2.4 修正后的提示词模板

```text
补充指令：如果你在本轮回复中生成了 Markdown 文件，请在文件保存完成后，执行以下命令用 IDEA 打开它。注意：不要修改脚本路径，只需把 `<Markdown文件绝对路径>` 替换为你实际生成的 Markdown 文件绝对路径。

powershell -ExecutionPolicy Bypass -File "{{scriptPath}}" -FilePath "<Markdown文件绝对路径>"
```

AIProcess 只需要替换 `{{scriptPath}}`，`<Markdown文件绝对路径>` 保留给模型替换。

### 2.5 修正后的追加函数

```ahk
AppendOpenMdPromptIfNeeded(content) {
    global AppConfig, TemplateDir
    if (!AppConfig["OpenMdWithIdea"]) {
        return content
    }

    templatePath := TemplateDir "\open_md_prompt.txt"
    if (!FileExist(templatePath)) {
        return content
    }

    template := FileRead(templatePath, "UTF-8")
    template := StrReplace(template, "{{scriptPath}}", AppConfig["OpenMdScriptPath"])

    return RTrim(content, "`r`n") "`r`n`r`n" template
}
```

三个入口统一调用 `AppendOpenMdPromptIfNeeded(content)`，不再传入具体 Markdown 路径。
