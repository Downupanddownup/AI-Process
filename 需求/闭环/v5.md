## 二、接受你的方案：用 PowerShell 脚本封装打开逻辑

你指出"每次让模型自己编写复杂命令不现实，容易出错"，这一点完全正确。直接让模型拼接 IDEA 路径 + 文件路径 + 引号转义，确实容易出错，尤其是路径中包含空格时。

因此改为：**提供一个 PowerShell 脚本 `OpenMarkdown.ps1`，由模型调用，脚本内部负责读取配置、校验路径、执行打开。**

## 三、脚本设计

### 3.1 脚本位置（测试版）

当前为验证而创建的测试脚本位于：

```text
C:\NotSystem\ClaudeCodeTool\AIProcess\需求\闭环\OpenMarkdown.ps1
```

### 3.2 脚本功能

- **接收参数**：`-FilePath`，要打开的 Markdown 文件绝对路径。
- **读取配置**：自动查找 `settings.ini`，读取 `[Editor]` 段下的 `IdeaCommand`。
- **校验**：
  - `settings.ini` 是否存在
  - `IdeaCommand` 是否配置
  - IDEA 可执行文件是否存在
  - 要打开的 Markdown 文件是否存在
- **执行**：调用 `Start-Process` 用 IDEA 打开文件。

### 3.3 脚本内容（测试版）

```powershell
param(
    [Parameter(Mandatory=$true, Position=0)]
    [string]$FilePath
)

$ErrorActionPreference = "Stop"

# 根据脚本所在位置自动寻找 settings.ini
$scriptDir = Split-Path -Parent $MyInvocation.MyCommand.Path
$candidatePaths = @(
    (Join-Path $scriptDir "..\..\app\config\settings.ini"),
    (Join-Path $scriptDir "..\app\config\settings.ini"),
    (Join-Path $scriptDir "config\settings.ini")
)

$configPath = $null
foreach ($candidate in $candidatePaths) {
    $resolved = Resolve-Path $candidate -ErrorAction SilentlyContinue
    if ($resolved -and (Test-Path $resolved)) {
        $configPath = $resolved
        break
    }
}

if (-not $configPath) {
    Write-Error "settings.ini not found"
    exit 1
}

# 解析 INI
$ini = @{}
$section = $null
foreach ($line in Get-Content $configPath) {
    $line = $line.Trim()
    if ([string]::IsNullOrWhiteSpace($line) -or $line.StartsWith(";")) {
        continue
    }
    if ($line -match "^\[(.+)\]$") {
        $section = $matches[1]
        $ini[$section] = @{}
    } elseif ($section -and $line -match "^(.+?)=(.*)$") {
        $ini[$section][$matches[1].Trim()] = $matches[2].Trim()
    }
}

$ideaCommand = $ini["Editor"]["IdeaCommand"]
if (-not $ideaCommand) {
    Write-Error "IdeaCommand not found"
    exit 1
}

if (-not (Test-Path $ideaCommand)) {
    Write-Error "IDEA executable not found: $ideaCommand"
    exit 1
}

if (-not (Test-Path $FilePath)) {
    Write-Error "Markdown file not found: $FilePath"
    exit 1
}

Start-Process -FilePath $ideaCommand -ArgumentList "`"$FilePath`"" -NoNewWindow
Write-Host "Opened $FilePath in IDEA"
```

## 五、AI 如何使用这个脚本

在提示词中，当"打开MD"开关开启时，追加如下指令：

```text
如果你在本轮回复中生成了 Markdown 文件（如 vN.md 或 实施文档.md），请在文件保存完成后，执行以下 PowerShell 命令用 IDEA 打开它（将 {{mdFilePath}} 替换为实际生成的文件绝对路径）：

powershell -ExecutionPolicy Bypass -File "{{scriptPath}}" -FilePath "{{mdFilePath}}"
```

其中：

- `{{scriptPath}}`：PowerShell 脚本的绝对路径，例如 `C:\NotSystem\ClaudeCodeTool\AIProcess\app\OpenMarkdown.ps1`。
- `{{mdFilePath}}`：AI 实际生成的 Markdown 文件路径。

这样 AI 只需要执行一条简单、固定的命令，不需要自己处理 IDEA 路径和引号转义。

## 六、正式脚本的放置位置

测试脚本暂时放在 `需求/闭环/` 目录。正式实施时，建议放到以下位置之一：

| 位置 | 优点 | 缺点 |
|---|---|---|
| `app/OpenMarkdown.ps1` | 与应用主程序同目录，运行时路径稳定 | 与 AHK 文件混在一起 |
| `script/OpenMarkdown.ps1` | 与安装脚本同目录，符合"脚本"语义 | 当前 `script/` 主要放安装/卸载脚本，可能混淆 |
| `app/bin/OpenMarkdown.ps1` | 独立目录，结构清晰 | 需要新增目录 |

**我的建议**：`app/OpenMarkdown.ps1`，因为该脚本属于应用运行时能力，与安装脚本职责不同。

## 七、仍需要确认的两个细节

1. **脚本最终放在 `app/` 还是 `script/`？**（我推荐 `app/`）
2. **是否需要在脚本中增加"如果 IDEA 已打开该文件，则不重复打开"的逻辑？**
   - 你之前认为不会重复打开，正常流程确实不会。
   - 但如果 AI 因为某种原因两次调用脚本打开同一个文件，IDEA 本身会处理（通常是激活已打开的 tab），所以脚本层面可以不处理。
