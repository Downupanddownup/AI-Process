## 二、当前核心工作流

1. **准备**：资源管理器右键目录 → "AIProcess 目录" → 启动并设置 `CurrentDir`。
2. **写需求**：在 `需求.txt` 中写需求。
3. **发任务**：按 `F2` 打开面板 → 点"建需求"或"复需求" → 提示词复制到剪贴板并自动发送给绑定的 Agent 窗口。
4. **看回复**：Agent 生成 `对vN的回复.txt`。
5. **迭代**：点"建回复"生成 `v(N+1).md`；点"复回复"继续沟通。
6. **定稿**：形成 `实施文档.md`。
7. **执行**：点"复执行"，Agent 按实施文档执行。

## 三、当前"用 IDEA 打开文件"机制

### 3.1 配置项

在 `app/config/settings.ini` 的 `[Editor]` 段：

```ini
[Editor]
OpenWithIdea=1
IdeaCommand=C:\NotSystem\app\idea\IntelliJ IDEA 2023.3.8\bin\idea64.exe
```

### 3.2 代码中相关定义

全局变量与控件：

```ahk
global OpenWithIdeaCheckbox := ""
global AppConfig := Map()
```

配置加载：

```ahk
AppConfig["OpenWithIdea"] := IniRead(SettingsFile, "Editor", "OpenWithIdea", "1") = "1"
AppConfig["IdeaCommand"] := IniRead(SettingsFile, "Editor", "IdeaCommand", "idea64.exe")
```

### 3.3 复用函数

```ahk
OpenFileInIdea(filePath) {
    global AppConfig
    try {
        Run('"' AppConfig["IdeaCommand"] '" "' filePath '"')
    } catch Error {
        ShowFeedback("IDEA 打开失败，请检查 settings.ini 中的 IdeaCommand", true)
    }
}
```

### 3.4 触发位置

目前以下 3 个地方会调用 `OpenFileInIdea(filePath)`：

| 触发点 | 操作 | 打开的文件 |
|---|---|---|
| 右键菜单启动后 | `SetCurrentDirAndOpenRequirement()` | `需求.txt` |
| "建需求"按钮 | `CreateRequirementFile()` | `需求.txt` |
| "建回复"按钮 | `CreateReplyFile()` | `对vN的回复.txt` |

### 3.5 UI 开关

主面板第一行复选框：

```ahk
OpenWithIdeaCheckbox := MainGui.AddCheckbox("xm y+6", "IDEA开")
OpenWithIdeaCheckbox.Value := AppConfig["OpenWithIdea"] ? 1 : 0
OpenWithIdeaCheckbox.OnEvent("Click", ToggleIdeaOpen)
```

切换时：

```ahk
ToggleIdeaOpen(ctrl, *) {
    global AppConfig, SettingsFile
    checked := ctrl.Value = 1
    AppConfig["OpenWithIdea"] := checked
    IniWrite(checked ? "1" : "0", SettingsFile, "Editor", "OpenWithIdea")
    ShowFeedback(checked ? "已开启 IDEA 打开" : "已关闭 IDEA 打开")
}
```

## 四、对本次需求的理解

### 4.1 需求原文

> 关于这个项目，你要好好看一下。我想到了一个绝妙的主意，可以用于让这个项目形成一个闭环。
>
> 好好研究一下，看怎么让 Idea 去打开一个 Markdown 文档。看当前这个快捷线上它是怎么去打开文件的，然后怎么去做那个配置的？解开关配置的。
>
> 我接下来需要追加一个开关，那就是用 Idea 打开 Markdown 或简化为打开 MD。这是用来完成整个快捷操作的最后一个环节了。也就是每次模型如果在回复时生成了一个 Markdown 文档，那么当它生成完成、结束之后，它需要通过我们给它提供的指令去打开 Markdown 文档。

### 4.2 我的理解

1. **目标**：给 AI Process 追加一个开关/配置，让 AI 在生成 Markdown 文件后，能够自动/按指令用 IDEA 打开它。
2. **本质**：把当前只打开 `.txt` 的能力，扩展到打开 `.md`。
3. **位置**：这是"快捷操作的最后一个环节"，意味着是在提示词层面给 AI 一个指令，让 AI 知道自己生成完 `.md` 后要调什么命令/操作。
4. **形式**：可能是一个新的复选框（如"打开 MD"），也可能是一个提示词模板中的占位符/指令。

## 五、可能的实现方案

### 方案 A：新增"打开 MD"复选框 + 配置项

在主面板增加一个复选框，例如：

- 文本：`打开MD`
- 配置键：`[Editor] OpenMdWithIdea`（或 `OpenMarkdown`）
- 默认：`1`

当勾选时，AI 生成的 Markdown 文件（如 `vN.md`、`实施文档.md`）创建后，调用 `OpenFileInIdea()` 打开。

**改动点预估**：

1. `EnsureDefaultFiles()`：初始化 `OpenMdWithIdea=1`。
2. `LoadConfig()`：读取 `OpenMdWithIdea`。
3. `CreateMainGui()`：新增 `OpenMdWithIdeaCheckbox` 控件与事件。
4. 新增 `ToggleOpenMdWithIdea()` 函数。
5. 在生成/保存 `.md` 文件的地方（如果有此类代码路径）调用 `OpenFileInIdea()`。

### 方案 B：提示词模板中增加"打开 Markdown"指令

在相关模板（如 `requirement_prompt.txt`、`reply_prompt.txt`、`execute/*.txt`）中追加占位符，例如：

```text
生成完成后，请用 IDEA 打开文件：{{mdFilePath}}
```

这样每次复制提示词时，AI 收到的指令里就包含"生成后打开"的操作。AI 自己生成完 Markdown 后，会按提示词去执行打开动作。

**改动点预估**：

1. 新增模板文件，例如 `open_md_prompt.txt`。
2. 在 `CopyRequirementPrompt()`、`CopyReplyPrompt()`、`CopyExecutePrompt()` 等函数中拼接该指令。
3. 通过 `AppConfig["OpenMdWithIdea"]` 控制是否拼接。

### 方案 C：A + B 结合

- 面板提供一个"打开MD"开关。
- 开关状态决定是否向 AI 的提示词中追加"生成后打开 Markdown"的指令。
- 同时，如果本地创建 `.md` 文件，也按开关决定是否用 IDEA 打开。

这是我认为**最完整**的方案，既能控制本地行为，也能把指令传给 AI。

- a) **并列开关**：`IDEA开` 管 `.txt`，`打开MD` 管 `.md`，互不影响。
- b) **细化开关**：`IDEA开` 是总开关，`打开MD` 是其子开关，只有总开关开启时子开关才生效。

## 七、我的初步建议

基于"闭环"和"最后一个环节"的描述，我建议采用 **方案 C（A + B 结合）**：

1. **面板新增"打开MD"复选框**，与"IDEA开"并列。
2. **配置项** `[Editor] OpenMdWithIdea=1`。
3. **本地行为**：当 AI Process 自己创建 `.md` 文件时（如未来可能的"建版本文档"按钮），按开关决定是否用 IDEA 打开。
4. **提示词行为**：在 `reply_prompt.txt` 等模板中追加占位符，当开关开启时，复制给 AI 的提示词里包含：

   ```text
   如果你生成了 Markdown 文件（如 vN.md 或 实施文档.md），生成完成后请用 IDEA 打开它：{{mdFilePath}}
   ```

5. **失败降级**：如果 IDEA 打开失败，可尝试用系统默认程序打开，避免流程卡死。
