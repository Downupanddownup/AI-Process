# 04-集成回归与收尾

## 目标

在目录规则、切换逻辑和 GUI 接入完成后，做集中回归验证，确保结果微调能力融入现有工具而不引入回归。

## 实施范围

1. 检查主题目录与问题子目录来回切换
2. 检查现有文件流程按钮在问题子目录中仍可用
3. 检查显隐状态与路径显示是否一致
4. 检查边界条件与错误处理

## 重点回归项

### 1. 目录切换

1. 主目录下点击 `建问题` 可进入 `结果微调\01`
2. 返回后再点击可进入 `结果微调\02`
3. `返` 可稳定返回主题目录

### 2. 现有能力复用

在问题子目录中：

1. `建需求` 仍正常创建 `需求.txt`
2. `复需求` 正常工作
3. `建回复` / `复回复` 正常工作
4. `复关系` 正常工作
5. `复执行` 正常工作

### 3. 边界情况

1. 未设置当前目录时，不应误触目录级动作
2. 普通非结果微调目录不应被误判为问题子目录
3. 状态标记不应在主目录态出现
4. 子目录态不应继续显示 `设目录`
5. 如果要切到其他目录，必须先 `返` 回主题目录

## 完成标准

本需求只有在以下全部满足时才算完成：

1. 主题目录与问题子目录切换闭环成立
2. 工具不在 `结果微调` 总目录停留
3. 现有文件流程完整复用
4. GUI 状态表达清晰且不拥挤
