# 03-GUI与状态提示

## 目标

将结果微调目录能力接入当前小面板 GUI，在有限空间内完成最小可理解表达。

## 实施范围

1. 在 GUI 中新增 `建问题` 按钮
2. 在 GUI 中新增 `返` 按钮
3. 增加子目录状态标记
4. 根据当前目录状态控制按钮显隐与标记显隐

## 具体内容

### 1. 按钮显示规则

#### 主题目录状态

1. 显示 `设目录`
2. 显示 `建问题`
3. 隐藏 `返`
4. 隐藏状态标记

#### 问题子目录状态

1. 隐藏 `设目录`
2. 隐藏 `建问题`
3. 显示 `返`
4. 显示状态标记

### 2. 状态标记

默认优先使用：

`↳`

如果在当前字体或控件下效果不理想，可退到：

1. `◆`
2. `▣`
3. `子`

### 3. 当前目录显示联动

在以下场景后，需要统一刷新状态：

1. 手动设目录
2. `建问题` 后
3. `返` 后

## 补充规则

1. `设目录` 与 `返` 采用同位置互斥显示
2. 子目录状态下如果要切到其他目录，必须先 `返`
3. 不在子目录状态下保留 `设目录`

## 非目标

本步骤不处理现有 `建需求 / 复需求 / 复执行` 的业务逻辑改造，只处理它们所在界面的状态表达。

## 产出结果

完成本步骤后，用户能直观看到当前是在主题目录还是问题子目录，并使用对应按钮。

## 验收点

1. 主目录态看到 `设目录` 和 `建问题`
2. 子目录态看不到 `设目录`
3. 子目录态看不到 `建问题`
4. 子目录态只看到 `返`
5. 子目录态能看到状态标记
6. 当前界面布局仍保持紧凑，不明显破坏现有风格
