# 关于「归档和仓库概念」的二次对齐（v2）

## 一、对 v1 回复的理解与确认

请确认以下理解是否准确：

### 1. 仓库识别规则
- 保持与现有 `ExtractProjectName()` 一致：以 `\需求\` 或 `\need\`（即 NED）的父级目录作为仓库。
- 仓库名 = 该项目/工程目录名。

### 2. 仓库添加方式
- 本期只做「添加仓库」，不做「删除/编辑仓库」。
- 流程：输入路径 → 校验是否符合仓库规则 → 符合则加入仓库列表，不符合打回。
- 校验规则：路径下必须存在 `需求/` 或 `need/` 子目录。

### 3. 历史索引重构（关键变更）
- **现状**：所有主题的历史索引混写在 `history/index/YYYY-MM-DD.jsonl` 中。
- **新方案**：按仓库拆分，每个仓库一个目录，目录下再按天写主题日志。
- 示例：`history/repositories/仓库名/YYYY-MM-DD.jsonl` 或 `history/仓库名/YYYY-MM-DD.jsonl`。
- 好处：归档时只需更新该主题所在仓库的日志，无需遍历全量索引。

### 4. 主界面改造
- 主面板仍显示「当前主题」（当前目录名）。
- 将原来的「当前：XXX」文字 + 点击弹出完整路径 MsgBox，升级为一个**二级弹窗**。
- 二级弹窗内容：
  - 顶部展示当前主题完整路径。
  - 下方支持「切换仓库」和「新增仓库」。
  - 选择仓库并确定后，弹窗关闭，并在原二级弹窗中展示该仓库下所有主题目录。
  - 用户可在主题目录列表中切换已有主题，或输入新主题名新建主题。

### 5. 归档功能位置
- 归档不放在主面板，而是放到**仓库管理**中。
- 仓库管理可通过托盘菜单进入。
- 主面板专注业务操作，管理操作集中到仓库管理窗口。

## 二、基于回复的细化设计草案

### 1. 数据模型草案

```
仓库（Repository）
  ├── id: 唯一标识（例如路径哈希或自增编号）
  ├── name: 仓库名称（默认取仓库根目录名）
  ├── rootPath: 仓库根目录绝对路径
  ├── nedName: 需求目录名（"需求" 或 "need"）
  └── nedPath: 仓库需求目录绝对路径（rootPath + nedName）
```

仓库列表建议持久化到独立文件，例如：
- `app/config/repositories.json` 或
- `app/config/repositories.ini`

不直接塞进 `settings.ini`，因为仓库是独立概念，未来可能扩展字段。

### 2. 历史索引新结构草案

```
history/
└── repositories/
    ├── {仓库A名称}/
    │   ├── 2025-07-01.jsonl
    │   ├── 2025-07-02.jsonl
    │   └── ...
    ├── {仓库B名称}/
    │   ├── 2025-07-01.jsonl
    │   └── ...
```

每行 JSONL 仍保持现有字段：
```json
{"time":"2025-07-02 10:00:00","window":"W1","theme":"主题名","source":"面板","path":"D:\\仓库\\需求\\主题名"}
```

### 3. 二级弹窗交互草案

弹窗标题：「设置主题」

布局：
```
当前主题：D:\\仓库A\\需求\\主题1

仓库：  [仓库A ▼]  [新增仓库]

主题目录：
┌─────────────────────────────┐
│ 主题1                         │
│ 主题2                         │
│ 主题3                         │
│ ...                          │
└─────────────────────────────┘

新主题名：[________________]  [新建并切换]

            [确定]  [取消]
```

行为：
- 切换仓库下拉框：列出所有已添加仓库。
- 新增仓库按钮：弹出小窗输入路径，校验后刷新下拉框。
- 主题目录列表：展示当前仓库 `需求/` 下所有一级目录（不含 `归档/`）。
- 双击或选中主题后点「确定」：切换到该主题。
- 输入新主题名点「新建并切换」：在仓库 `需求/` 下创建新目录并进入。

### 4. 仓库管理窗口草案

入口：托盘菜单增加「仓库管理」。

布局：
```
┌──────────────┬──────────────────────────────┐
│ 仓库列表      │  主题目录列表                 │
│  - 仓库A      │  [ ] 主题1                    │
│  - 仓库B      │  [ ] 主题2                    │
│              │  [ ] 主题3                    │
│ [新增仓库]    │                              │
│              │  [归档选中主题]               │
└──────────────┴──────────────────────────────┘
```

行为：
- 左侧点击仓库，右侧展示该仓库下所有一级主题目录。
- 右侧支持多选（复选框）。
- 点击「归档选中主题」：将选中目录移入 `仓库/需求/归档/`，重名自动加序号。
- 归档完成后，同步更新该仓库的历史索引路径。

### 5. 归档规则草案

- 归档目录：`{仓库根目录}\需求\归档\`
- 若不存在则自动创建。
- 重名处理：`主题名_1`、`主题名_2` …（或带日期，待确认）。
- 归档后，主题路径从 `仓库\需求\主题A` 变为 `仓库\需求\归档\主题A`。
- 同步更新该仓库历史索引中所有 `主题A` 相关记录的 `path` 字段。

## 三、仍需确认的问题

### 1. 历史索引迁移

- 如果两个仓库根目录名相同但路径不同（例如 `D:\项目A\需求` 和 `E:\项目A\需求`），历史索引目录用仓库名会冲突。

- 校验时是否要求路径下必须同时存在 `需求.txt` 或已有主题？还是只要求存在 `需求/` 或 `need/` 目录即可？

### 4. 当前「设目录」按钮是否保留

### 5. 二级弹窗中「新增仓库」的交互

### 6. 归档重名规则

  - A. `主题名_1`、`主题名_2` …

  - C. 其他规则？

### 7. 是否支持「取消归档」
- 仓库管理中是否需要提供「取消归档」功能，将主题从 `归档/` 移回 `需求/` 根目录？
