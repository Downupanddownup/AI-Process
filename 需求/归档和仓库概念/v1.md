# 关于「归档和仓库概念」需求的对齐与问题

## 一、需求理解

从需求语音中提炼出以下核心诉求：

1. **仓库（Repository）概念**
   - 以 `需求/` 或 `need/` 目录的**上一级目录**作为仓库。
   - 仓库名即该项目/工程目录名。
   - 仓库相对稳定，主题每天都在建，但仓库数量较少。

2. **主题目录管理**
   - 在软件中先选择仓库。
   - 然后可以选择已有主题目录，或新建主题目录。
   - 避免每次都手动去资源管理器里建文件夹、复制路径。

3. **归档（Archive）功能**
   - 在仓库的 `需求/` 目录下创建一个**归档**文件夹。
   - 列出该仓库下所有一级主题目录，支持勾选。
   - 将选中的主题目录批量移入归档文件夹。
   - 若发生重名，自动重命名以避免冲突。

4. **归档后数据不丢失**
   - 日志（`.aiprocess/log.jsonl`）跟着主题目录走。
   - `history/index/*.jsonl` 仍能定位到主题。
   - 周报、经验总结仍然可以正常生成。

## 二、项目现状核对

基于当前代码（`app/` 目录）逐项核对：

| 需求项 | 当前是否已实现 | 说明 |
|---|---|---|
| 仓库概念 | ❌ 未实现 | 代码中无仓库对象、仓库列表、仓库管理 UI。仅有 `ThemePathUtils.ahk` 中的 `ExtractProjectName()` 从路径中提取「归属项目」。 |
| 归档功能 | ❌ 未实现 | 无归档目录创建、批量移动、重命名冲突处理。 |
| 主题目录管理 | ⚠️ 部分实现 | `ThemeManager.ahk` 支持新建主题，`DirectoryManager.ahk` 支持手动设置当前主题目录，但都是在「主题」层级，没有「仓库 → 主题」的二级选择。 |
| 历史索引 | ✅ 已实现 | `history/index/*.jsonl` 记录进入目录的路径，经验总结窗口据此加载主题。 |
| 经验总结窗口 | ✅ 已实现 | `SummaryWindow.ahk` 可展示主题列表、归属项目、最后访问时间、总结状态、目录状态。 |
| 周报/报告 | ✅ 已实现 | `ReportGenerator.ahk` 基于历史索引和主题目录生成周报/日报。 |
| 多窗口配置 | ✅ 已实现 | `settings.ini` 中有 `Window1/2/3` 三个独立 `CurrentDir` 配置。 |

**关键发现**：
- 经验总结和周报都依赖 `history/index/*.jsonl` 中的 `path` 字段。
- 归档意味着主题路径会变化（从 `仓库/需求/主题A` 变为 `仓库/需求/归档/主题A`）。
- 如果历史索引中的旧路径不更新，现有 `FindThemeDir()` 逻辑需要能够**从归档路径反向定位主题根目录**，否则会出现「目录状态」异常或无法生成总结。

## 三、待确认问题

以下问题无法仅通过现有代码推断，需要明确：

### 1. 仓库识别规则
- 是否固定以 `\需求\` 或 `\need\` 的父级目录作为仓库？（与现有 `ExtractProjectName()` 规则保持一致？）
- 是否允许用户手动添加/移除仓库？还是完全自动发现硬盘上的所有 `*/需求/` 或 `*/need/` 目录？
- 仓库列表需要持久化保存吗？例如存到 `settings.ini` 还是独立配置文件？

### 2. 主界面流程改造
- 是否将现有「设目录」按钮改造为「选仓库 → 选主题」的两级流程？
- 还是保留现有「设目录」，另外新增一个独立的「仓库管理」窗口？
- 当前 `Window1/2/3` 各有一个 `CurrentDir`，切换仓库时，是三个窗口独立绑定不同仓库，还是全局统一仓库？

### 3. 归档目录与命名
- 归档文件夹名称固定为 `归档`，还是允许自定义？
- 重名冲突时的重命名规则建议：
  - 方案 A：`主题A_1`、`主题A_2` …
  - 方案 B：`主题A_20250702`、`主题A_20250702_1` …
  - 你倾向哪种？
- 是否支持多级归档，例如按 `归档/2025/` 或 `归档/已完成/` 分组？

### 4. 历史索引同步
- 主题归档后，是否需要更新 `history/index/*.jsonl` 中所有涉及该主题的旧路径？
- 还是保留旧路径，让经验总结窗口通过「主题名 + 仓库名」重新扫描定位？
- 若采用「不更新索引，仅扫描定位」方案，已归档主题在列表中如何标识？（例如目录状态显示为「已归档」）

### 5. 归档后的可操作性
- 已归档主题是否仍能继续工作（创建需求、生成总结、生成报告）？
- 是否需要提供「取消归档」功能，即从 `归档/` 目录移回原位？
- 周报/日报生成时，是否默认包含已归档主题？是否需要「包含归档」开关？

### 6. 新建主题的位置
- 选择仓库后，新建主题是默认在 `仓库/需求/` 下直接创建，还是允许选择 `仓库/需求/` 下的某个子文件夹？
- 当前 `CreateNewTheme()` 是在当前主题目录的**父级**创建新主题。改造后是否需要改为在仓库的 `需求/` 目录下创建？

### 7. 已有历史数据
- 当前硬盘上已存在大量主题目录和 `history/index/*.jsonl` 记录。
- 首次启用仓库功能时，是否需要自动将历史主题归入对应的仓库？还是仅对未来新建/归档生效？

## 四、初步设想（待确认后细化）

若上述问题确认后，可大致按以下方向实现：

1. **仓库模型**
   - 每个仓库 = `仓库根目录 + 需求目录名（需求/need）`。
   - 仓库列表持久化到 `app/config/repositories.ini` 或 `settings.ini` 的新段。

2. **仓库管理窗口**
   - 新增「仓库管理」窗口：左侧仓库列表，右侧该仓库下主题列表（一级目录）。
   - 支持勾选主题 → 点击「归档」→ 移入 `仓库/需求/归档/`。

3. **主界面集成**
   - 「设目录」升级为：先选仓库，再选主题。
   - 或保留「设目录」，新增「仓库」按钮专门用于仓库/主题选择。

4. **索引同步**
   - 归档时同步更新 `history/index/*.jsonl` 中该主题的旧路径为新路径。
   - 这样经验总结和周报无需额外处理即可兼容。

5. **标识与筛选**
   - 经验总结窗口增加「目录状态」值：`正常` / `已归档`。
   - 报告生成窗口增加「包含归档」复选框。
