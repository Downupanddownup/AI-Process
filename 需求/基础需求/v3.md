# V3 想法与问题整理

## 当前结论

这一轮之后，产品范围已经基本确定，前面那些偏发散的设想可以正式收掉。

目前已经明确的点有：

1. 主题目录由你自己创建。
2. 工具只负责保存和切换当前目录路径。
3. 固定文件名使用 `需求.txt`。
4. 回复类按钮复制的是：`文件路径 + 一段固定提示词`。
5. 第一版不负责自动发送给 AI，只复制到剪贴板。
6. 如果文件已存在，就提示“文件已存在”。
7. 如果“创建后用 IDEA 打开”开关已开启，则文件存在时也执行打开动作。

这说明第一版已经可以进入实施设计了。


## 这轮之后我认为还值得补的想法

### 1. “对VX的回复.txt” 需要明确自动取哪个版本

你已经明确说了，回复文件名肯定由程序自动生成，这个我认同。

那现在剩下的问题不是“要不要自动生成”，而是：

`程序依据什么判断当前应该生成 对v1的回复.txt 还是 对v2的回复.txt`

这个规则必须写死清楚。

我建议第一版直接按当前目录内已有文件来判断：

1. 如果存在 `v1.md`，但不存在 `对v1的回复.txt`，就生成 `对v1的回复.txt`
2. 如果存在 `v2.md`，但不存在 `对v2的回复.txt`，就生成 `对v2的回复.txt`
3. 优先处理“最新一个还没有回复文件的 vX.md”

这样你就不需要额外输入，也符合你的真实使用习惯。


### 2. “复制回复提示词”也需要明确取哪个回复文件

和上一条一样，创建按钮和复制按钮都要有一致规则。

例如：

1. 点击“创建回复文件”时，定位最新未回复的 `vX.md`
2. 点击“复制回复提示词”时，优先定位最新的 `对vX的回复.txt`

否则会出现一个问题：

目录里已经有 `对v1的回复.txt` 和 `对v2的回复.txt`，此时复制按钮到底复制哪一个？

这个也最好明确成固定规则，不要交给临时猜测。


### 3. 需要考虑“按钮不可执行”的提示

这个是个小点，但很实际。

例如：

1. 当前还没设置目录路径
2. 当前目录下还没有 `v1.md`
3. 你点击了“创建对VX回复文件”

这时候不能静默失败，最好直接给出明确提示，比如：

1. `请先设置当前主题目录`
2. `当前目录下未找到可回复的 vX.md 文件`

这会让工具稳定很多。


### 4. “文件关系说明复制按钮”最好尽快固化文本内容

你已经确认这个按钮有价值，那它下一步不是继续讨论功能，而是尽快把那段标准说明文案定下来。

因为这个按钮本质上不是程序逻辑问题，而是内容模板问题。

只要这段说明模板定下来，后面实现会非常简单。

我建议它至少说明这几件事：

1. `需求.txt` 是原始需求说明
2. `v1.md`、`v2.md` 是 AI 的问题和想法迭代文档
3. `对vX的回复.txt` 是你对对应版本文档的回复
4. `实施文档.md` 是需求收敛后的最终执行依据
5. 如果当前还在需求澄清阶段，禁止直接改代码和正式文件


## 现在还剩下的少量关键问题

### 1. 最新版本的判断规则是否按“最大 vX 编号”处理

这是我现在最想确认的实现问题。

例如目录里有：

1. `v1.md`
2. `对v1的回复.txt`
3. `v2.md`

那程序显然应该处理 `v2.md`。

但如果目录里有：

1. `v1.md`
2. `v3.md`

中间缺了 `v2.md`，程序是：

1. 按最大编号 `v3.md` 处理
2. 还是按最早未完成编号处理

我建议第一版直接按：

`当前目录中编号最大的 vX.md 作为当前版本`

逻辑最简单。


### 2. 创建 `需求.txt` 时，是否需要自动写入初始模板内容

你现在只说了创建文件和可选打开，但这里还有一个小优化空间：

新建 `需求.txt` 后，是否要自动写入一段初始占位内容。

例如：

1. 当前问题背景
2. 当前现状
3. 我的目标
4. 我想到的方案

如果要，这个按钮会更实用。

如果不要，那就只是创建空文件。

这个点会影响第一版体验，但实现不复杂。


### 3. “定位切换”是否需要保留最近一次路径

你当前描述的是手动填路径切换上下文。

那还剩一个小问题：

关闭工具后，下次打开是否保留上一次路径。

如果保留，使用体验会更顺。
如果不保留，每次都要重新贴路径。

我倾向于：

`默认保留最近一次成功设置的路径`


## 结论

这一轮之后，已经不是要不要写 `v4` 的问题了，而是基本可以开始写实施文档了。

如果还要写 `v3`，我认为它的价值主要就在这三件事：

1. 明确 `vX` 和 `对vX回复` 的自动判定规则
2. 明确按钮不可执行时的提示逻辑
3. 尽快固化“文件关系说明”的标准文案

除此之外，已经没有新的大方向问题了。
