# 经验窗口的主题总结优化 —— 实施文档

## 一、需求边界

### 1.1 目标

改造「经验总结」功能，使 AI Agent 从直接生成 Markdown 总结，改为生成分层的结构化 JSON 计算文件。后续 Markdown 视图由本地脚本按需从 JSON 渲染。

### 1.2 最终产物

| 产物 | 位置 | 说明 |
|---|---|---|
| `Summary.json` | `{themePath}\.aiprocess\Summary.json` | AI 生成的唯一计算文件 |
| `Summary.md` | `{themePath}\.aiprocess\Summary.md` | 用户点击「查看总结」时由脚本从 JSON 渲染生成 |
| 稳定规则文件 | `app/templates/summary/` | `summary_input_guide.md`、`summary_template.json`、`summary_rules.md` |
| 本地脚本 | `app/powershell/summary/` | `PrepareSummaryInput.ps1`、`ConvertSummaryToMarkdown.ps1` |

### 1.3 不变更范围

- 不改动「经验总结」窗口的整体 UI 布局。
- 不改动 Agent 绑定、发送机制。
- 不改动报告（周报/日报）生成逻辑。
- 不改动历史索引、操作日志写入逻辑。

### 1.4 输入输出约定

**输入（本地脚本生成）**：
- `summary_input.json`：主题轮次、时间、字数、内容预览等基础素材。

**输入（稳定规则文件）**：
- `summary_input_guide.md`：解释 `summary_input.json` 字段含义。
- `summary_template.json`：规定输出 JSON 的完整结构。
- `summary_rules.md`：规定 AI 如何分类、摘要、分析、自检。

**输出**：
- `Summary.json`：AI 生成的结构化总结。
- `Summary.md`：用户查看时由 `ConvertSummaryToMarkdown.ps1` 从 JSON 渲染。

## 二、轮次与时间定义

### 2.1 轮次定义

一轮 = 用户创建的 `.txt` 文件 → AI 创建的 `.md` 文件。

- 第 0 轮：`需求.txt` → `v1.md`
- 第 1 轮：`对v1的回复.txt` → `v2.md`
- 第 N 轮：`对vN的回复.txt` → `v(N+1).md`

`实施文档.md` 本身不是一轮，而是讨论阶段收敛的产物。`结果微调/` 子目录内部按同样规则重新构成轮次。

### 2.2 时间计算规则

沿用原项目的「活跃时长」逻辑：

- 从主题最早文件/日志时间到最后文件修改时间，作为总时间范围。
- 相邻活跃点（文件创建、日志记录）之间超过 `IdleThresholdMinutes`（默认 60 分钟）的间隔，不计入活跃时长。
- 每轮耗时 = 该轮人输入文件创建时间 → AI 输出文件创建时间，按分钟向上取整。
- 优先使用 `log.jsonl` 时间；没有日志时回退到文件创建/修改时间。

### 4.2 修改文件

| 文件 | 修改内容 |

## 五、实现步骤

### 步骤 1：新增稳定规则文件

将以下三份文件放入 `app/templates/summary/`：

1. `summary_input_guide.md`
2. `summary_template.json`
3. `summary_rules.md`

内容基于「闭环」验证时 `_tmp` 下的草稿定稿。

### 步骤 2：新增 `PrepareSummaryInput.ps1`

输入：`-ThemePath <path>`
输出：`{themePath}\.aiprocess\_tmp\summary_input.json`

功能：

1. 扫描主题目录，识别 `需求.txt`、`v*.md`、`对v*.txt`。
2. 按轮次规则配对。
3. 计算每轮起止时间、耗时、字数。
4. 提取每轮内容预览（前 200 字）。
5. 复用 `ActiveDurationCalculator.ps1` 的活跃时长逻辑。
6. 输出 `summary_input.json`。

### 步骤 3：修改 `SummaryDataCollector.ahk`

- 将 `psScript` 从 `PrepareSummaryData.ps1` 改为 `PrepareSummaryInput.ps1`。
- 输出文件参数可保持不变，脚本内部决定写入 `summary_input.json`。
- AHK 读取后解析 JSON 返回 Map。

### 步骤 4：修改 `SummaryGenerator.ahk`

- `GenerateSummary()`：
  - 校验 `Summary.json` 是否已存在（替代 `Summary.md`）。
  - 调用 `CollectSummaryData(themePath)` 获取素材。
  - 将 `summary_input.json` 和三份规则文件复制/引用到 `{themePath}\.aiprocess\_tmp\`。
  - 构造 Agent 短消息。
- `BuildSummaryPrompt()`：
  - 废弃旧模板读取逻辑。
  - 返回包含素材路径和规则路径的短消息。

### 步骤 5：新增 `ConvertSummaryToMarkdown.ps1`

输入：`-JsonPath <path>`
输出：同目录 `Summary.md`

功能：

1. 读取 `Summary.json`。
2. 按固定模板渲染为 Markdown。
3. 写入 `{themePath}\.aiprocess\Summary.md`。

### 步骤 6：修改 `SummaryWindow.ahk`

- `ViewThemeSummary(path)`：
  1. 若 `Summary.md` 存在，直接打开。
  2. 若 `Summary.md` 不存在但 `Summary.json` 存在：
     - 调用 `ConvertSummaryToMarkdown.ps1`。
     - 生成后打开 `Summary.md`。
  3. 若都不存在，提示未生成。
- `RenderThemeList()`：
  - `summaryExists` 判断改为 `FileExist(Summary.md) || FileExist(Summary.json)`。

### 步骤 7：修改 `NotifySummaryComplete.ps1`

- Agent 生成 `Summary.json` 后调用此脚本。
- 校验 `Summary.json` 存在。
- 不再要求 `Summary.md` 必须存在。
- 写 `history/summary_log.jsonl`。
- 发送 `0x8000` 刷新窗口。

### 步骤 8：联调测试

1. 在「闭环」主题上点击「生成总结」。
2. 确认 Agent 收到正确消息并生成 `Summary.json`。
3. 点击「查看总结」，确认 Markdown 被渲染并打开。
4. 在另一个主题（如「历史财富」）上重复测试。

### 步骤 9：清理旧文件

新功能稳定运行后，删除以下旧文件：

- `app/templates/summary/summary_prompt.txt`
- `app/templates/summary/summary_template.md`
- `app/powershell/summary/PrepareSummaryData.ps1`

## 六、关键约束

1. **模块划分**：
   - PowerShell 脚本只负责数据处理和格式转换，不依赖 AHK 业务状态。
   - AHK 业务模块调用 PowerShell，但不把业务逻辑写入 PowerShell。
   - 模板文件只包含规则/结构，不包含业务调用代码。

2. **单向依赖**：
   - `PrepareSummaryInput.ps1` 不依赖 `SummaryGenerator.ahk`。
   - `ConvertSummaryToMarkdown.ps1` 只读取 JSON，不依赖生成过程。

3. **兼容过渡**：
   - 旧文件在新功能稳定前保留。
   - 旧 Prompt/模板不再被调用。

4. **错误处理**：
   - `PrepareSummaryInput.ps1` 失败时返回错误信息给 AHK。
   - `ConvertSummaryToMarkdown.ps1` 失败时提示用户，不崩溃。
   - Agent 未按规则生成 JSON 时，由用户人工介入（后续可加强校验）。

---

## 七、验收标准

1. 点击「生成总结」后，Agent 能基于 `summary_input.json` + 三份规则文件生成合法 `Summary.json`。
2. `Summary.json` 包含完整字段：`rounds`、`overview`、`metrics`、`analysis`、`subThemes`。
3. 点击「查看总结」时：
   - 已有 `Summary.md` 直接打开。
   - 无 `Summary.md` 但有 `Summary.json` 时，自动生成 `Summary.md` 并打开。
4. 经验总结窗口的「总结状态」正确显示已总结/未总结。
5. 新流程不破坏报告生成、历史索引、操作日志等其他功能。
