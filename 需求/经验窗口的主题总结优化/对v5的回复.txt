关于生成的内容，我想了一下，这些内容最终我们应该做成这样：

1. 明确阶段划分
   我们让 AI Agent 去做的时候，分成以下几个阶段：
   (a) 尽量把那些可以通过脚本确定的内容，全部通过代码生成好、补全好，并以分门别类、最友好、最易读的方式整理好素材。
   (b) 准备一份文件，说明为它准备了什么样的素材、这些素材是什么样的含义。
   (c) 最终需要它生成一个什么样的 JSON 文件。
   (d) 明确这些 JSON 文件当中，哪些内容是需要它自己去生成的，哪些内容则不需要它自己生成。
   (e) 明确需要它生成的一些内容里，应该按照什么样的规则去生成。
   (f) 明确最终它是如何去校验生成的 JSON 文件是否符合标准。

2. 制定规则文档
   我需要把通过程序生成的内容跟要由 AI 生成的内容区分开。对于 AI Agent 的生成内容，我们要给它定义一份规则文档。这个文档其实包含两个部分：
   (a) 生成最终的格式：即一个 JSON 的模板。
   (b) 内容提取与归类：比如需要有几个轮次的文档内容提取、整体内容的总结、章节内容名称的归类以及阶段的归类。这些内容我们需要有一份详细的说明文档，并与 JSON 结构进行对应。

3. 设计后续功能
   最终让 AI Agent 去生成这样的一份 JSON 文件。后面我们要做的事情主要有两件：
   (a) 设计一个基于 JSON 文件去生成 Markdown 文档的功能脚本。
   (b) 基于这个 JSON 文件去展示 HTML 内容的页面，以及将这个 JSON 文件和 HTML 页面关联在一起的快捷方式和启动方式。

你理解我的意思吧。其实就是 AI Agent 只需要生成这个 JSON 文件，然后后面基于这份文件怎么去转化为 Markdown 文档或者怎么去转化成 HTML 网页，我们通过程序去搞定。
