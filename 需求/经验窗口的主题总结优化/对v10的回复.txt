读完之后，你告诉我说，从技术角度来讲，我们要达成这个需求，当前的这个生成任务的函数要进行哪些改造？这个改造就只考虑 Markdown 文件的。

具体流程如下：

1. 我们点击生成按钮，然后去把那些素材（需要脚本生成素材）生成好。
2. 把哪些固定规则的文档和模板也放好，生成结果的模板也放好。
3. 一起发给 AI Agent，然后告诉他要怎么做。
4. AI Agent 处理之后，它会生成一个 JSON 文件。
5. 到时候你现在有一个功能叫“查看总结”，这个功能原本是查看一下 Markdown 文件。那现在的话你就改一下，改成：
   (a) 如果有 Markdown 文件，那你就直接读取显示的 Markdown 文件；
   (b) 如果它没有 Markdown 文件，那你就去找那个 JSON 文件。
6. 你再找设计一个脚本，先设计一个简单的 JSON 脚本，把这个 JSON 文件转换成 Markdown 文件。样式什么的先随便一点，我们后面再考虑具体怎么做。但是现在只把这流程走通。
7. 你把生成 Markdown 文件也放在原本那个主题目录下面（也就是 .api-process 目录下面），然后你再去打开这个 Markdown 文件。
