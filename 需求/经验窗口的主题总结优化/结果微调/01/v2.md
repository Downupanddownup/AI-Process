# 回复确认：无需提前删除，完成脚本会统一清理

## 你的理解是对的

当 Agent 生成完成后，会调用 `NotifySummaryComplete.ps1`。该脚本在第 81–93 行有明确的临时目录清理逻辑：

```powershell
# 4. 安全清空主题临时目录（仅删除 {ThemePath}\.aiprocess\_tmp 下的内容，不越界）
$tmpDir = Join-Path $ThemePath '.aiprocess\_tmp'
...
Remove-Item -LiteralPath $item.FullName -Recurse -Force -ErrorAction SilentlyContinue
```

也就是说，`_tmp` 下的 `summary_input.json`、`summary_prompt_*.txt`、三份规则模板文件等，最终都会由 `NotifySummaryComplete.ps1` 统一清理。

因此，`SummaryDataCollector.ahk` 在解析成功后立即删除 `summary_input.json` 是**多余且有害的**：

- 多余：后续有统一清理逻辑。
- 有害：Agent 提示词明确引用该文件，删除后 Agent 无法读取素材。

## 建议的精确修改

文件：`app\modules\business\summary\SummaryDataCollector.ahk`

### 修改 1：删除解析成功后的立即删除

当前第 58–66 行：

```autohotkey
try {
    content := FileRead(outputFile, "UTF-8")
    ; 去除可能的 BOM
    if (SubStr(content, 1, 1) = "`uFEFF") {
        content := SubStr(content, 2)
    }
    data := JSON.Load(content)
    FileDelete(outputFile)        ; ← 删除这一行
    return data
}
```

改为：

```autohotkey
try {
    content := FileRead(outputFile, "UTF-8")
    ; 去除可能的 BOM
    if (SubStr(content, 1, 1) = "`uFEFF") {
        content := SubStr(content, 2)
    }
    data := JSON.Load(content)
    return data
}
```

### 修改 2：解析失败时保留文件便于排查

当前第 67–73 行：

```autohotkey
catch Error as err {
    try {
        FileDelete(outputFile)    ; ← 建议删除这一行
    } catch {
        ; 忽略删除失败
    }
    return Map("Error", "解析数据失败：" err.Message)
}
```

改为：

```autohotkey
catch Error as err {
    return Map("Error", "解析数据失败：" err.Message)
}
```

保留 `summary_input.json` 可以在解析失败时直接人工检查 JSON 内容，排查是脚本生成问题还是解析问题。

## 验证计划

修复后建议在「周报时间」主题重新点击「生成总结」，验证：

1. `_tmp\summary_input.json` 在 Agent 执行期间保持存在。
2. Agent 成功读取并生成 `Summary.json`。
3. 生成完成后 `_tmp` 目录被 `NotifySummaryComplete.ps1` 清空。
4. 「查看总结」能正常从 `Summary.json` 渲染出 `Summary.md`。

## 结论

问题根因已确认：临时文件被提前删除。修复方案是**移除 `SummaryDataCollector.ahk` 中的 `FileDelete(outputFile)` 调用**，由 `NotifySummaryComplete.ps1` 统一负责 `_tmp` 清理。
