# 问题分析：summary_input.json 在 Agent 读取前被删除

## 现象

在「周报时间」主题点击「生成总结」后，`.aiprocess\_tmp\` 目录下：

- 存在 `summary_prompt_20260705223528188.txt`
- 存在 `summary_input_guide.md`、`summary_template.json`、`summary_rules.md`
- **不存在 `summary_input.json`**
- `.aiprocess\` 根目录下也未生成 `Summary.json`

即：基础输入文件确实没有出现，导致 Agent 无法基于素材生成总结。

## 根因定位

问题出在 `app\modules\business\summary\SummaryDataCollector.ahk` 的数据处理流程。

该模块当前逻辑：

1. 调用 `PrepareSummaryInput.ps1` 生成 `summary_input.json`。
2. 读取 `summary_input.json` 内容。
3. 用 `JSON.Load(content)` 解析为 AHK Map。
4. **成功后立即 `FileDelete(outputFile)` 删除 `summary_input.json`。**
5. 将解析后的 Map 返回给 `SummaryGenerator.ahk`。

而 `SummaryGenerator.ahk` 在拿到数据后，会构造给 Agent 的提示词，其中明确指示 Agent 读取：

```text
4. 主题素材：C:\...\周报时间\.aiprocess\_tmp\summary_input.json
```

也就是说，**基础输入文件在 Agent 读取之前就被本地模块删除了**。这是本次端到端失败的根本原因。

## 验证

- 提示文件 `summary_prompt_... .txt` 已生成，说明 `SummaryGenerator.ahk` 成功执行到了构造提示词阶段。
- `Summary.json` 未生成，说明 Agent 没有成功产出结果（因为引用的素材文件已不存在）。
- `_tmp` 目录中其他文件均保留，唯独 `summary_input.json` 缺失，与 `SummaryDataCollector.ahk` 中 `FileDelete(outputFile)` 的行为一致。

## 修复建议

删除 `SummaryDataCollector.ahk` 中两处对 `summary_input.json` 的主动删除逻辑：

- 第 65 行：解析成功后的 `FileDelete(outputFile)`
- 第 69 行：解析失败 catch 块中的 `FileDelete(outputFile)`（失败时保留便于排查）

`summary_input.json` 作为临时文件，最终应随 `_tmp` 目录一起由 `NotifySummaryComplete.ps1` 清理，而不是在数据解析成功后立即删除。

3. 修复后是否需要重新在「周报时间」主题上做端到端验证？
