# 实施文档：主题总结输入结构完善

## 一、需求背景

当前 `PrepareSummaryInput.ps1` 生成的 `summary_input.json` 存在以下问题：

1. `humanContentPreview` / `aiContentPreview` 由脚本截断原文生成，易造成 AI 误读，应删除。
2. `实施文档.md` 未作为轮次纳入，遗漏了「讨论 → 执行」的转折点。
3. `结果微调/XX/` 子目录下的轮次未被扫描，导致结果微调内容丢失。
4. `实施步骤/XX/` 子目录（即子主题）下的轮次未被扫描，导致复杂主题的拆分讨论丢失。
5. 总体时间、字数、指标只计算根目录轮次，无法反映主题真实全貌。

## 二、需求边界

### 2.1 本次必须完成

- 删除 `summary_input.json` 中的 `humanContentPreview` / `aiContentPreview` 字段。
- 将 `实施文档.md` 作为普通轮次加入 `rounds.items`，分类为 `"实施确认"`。
- 扫描主题根目录下的 `结果微调/XX/` 子目录，每个子目录作为一个结果微调单元，输出到 `resultFineTunings`。
- 扫描主题根目录下的 `实施步骤/XX/` 子目录，每个子目录作为一个子主题单元，输出到 `subThemes`。
- 每个子单元（结果微调/子主题）内部同样处理其 `实施文档.md` 作为「实施确认」轮次。
- `totalTime`、`totalHumanChars`、`totalAiChars`、轮次分类统计等按根目录 + 结果微调 + 子主题全量合并计算。
- 新增 `breakdown` 字段，拆分主讨论、结果微调、子主题三部分的轮次数、时长、字数贡献。
- 同步更新 `summary_input_guide.md`、`summary_template.json`、`summary_rules.md`。

### 2.2 本次不碰

- 不修改 Agent 调用流程（`SummaryGenerator.ahk`、`AgentDispatcherSend`）。
- 不修改 `Summary.json` → `Summary.md` 的渲染逻辑（`ConvertSummaryToMarkdown.ps1`）。
- 不修改 UI 层（`SummaryWindow.ahk`）。
- 不修改完成通知脚本（`NotifySummaryComplete.ps1`）。
- 不删除旧版废弃文件（待新流程稳定后统一处理）。

### 3.2 `PrepareSummaryInput.ps1` 改造细节

#### 3.2.1 删除预览字段

#### 3.2.2 新增「实施确认」轮次生成函数

新建函数 `Add-ImplementationRound`：

```powershell
function Add-ImplementationRound {
    param(
        [string]$Dir,
        [array]$Rounds,
        [array]$LogEntries
    )
    $implDoc = Join-Path $Dir '实施文档.md'
    if (-not (Test-Path $implDoc)) {
        return $Rounds
    }

    $lastRound = $Rounds | Sort-Object roundIndex | Select-Object -Last 1
    $lastHumanFile = if ($lastRound) { $lastRound.humanFile } else { '' }

    $humanTime = if ($lastRound) {
        [datetime]::ParseExact($lastRound.endTime, 'yyyy-MM-dd HH:mm:ss', $null)
    } else {
        (Get-Item $implDoc).CreationTime
    }

    $aiTime = Get-LogTimeForFile -LogEntries $LogEntries -FileName '实施文档.md' -FallbackTime (Get-Item $implDoc).CreationTime

    if ($humanTime -gt $aiTime) {
        $humanTime = $aiTime.AddSeconds(-1)
    }

    $duration = [math]::Ceiling(($aiTime - $humanTime).TotalMinutes)
    if ($duration -lt 1) { $duration = 1 }

    $newIndex = if ($lastRound) { $lastRound.roundIndex + 1 } else { 0 }

    $newRound = [PSCustomObject]@{
        roundIndex      = $newIndex
        stage           = '实施确认'
        humanFile       = $lastHumanFile
        aiFile          = '实施文档.md'
        startTime       = $humanTime.ToString('yyyy-MM-dd HH:mm:ss')
        endTime         = $aiTime.ToString('yyyy-MM-dd HH:mm:ss')
        durationMinutes = $duration
        humanChars      = 0
        aiChars         = Get-CharCount -Path $implDoc
        category        = '实施确认'
        isExtreme       = $false
        extremeReason   = ''
    }

    return $Rounds + @($newRound)
}
```

> 原则：新增独立函数，不在 `Get-ThemeRounds` 里追加不相关逻辑。

#### 3.2.3 新增子目录扫描函数

新建函数 `Get-SubUnits`：

```powershell
function Get-SubUnits {
    param(
        [string]$Dir,
        [string]$SubDirName,
        [array]$LogEntries
    )
    $subUnits = @()
    $subRoot = Join-Path $Dir $SubDirName
    if (-not (Test-Path $subRoot -PathType Container)) {
        return $subUnits
    }

    $childDirs = Get-ChildItem -LiteralPath $subRoot -Directory -Force | Sort-Object Name
    foreach ($child in $childDirs) {
        $requirementFile = Join-Path $child.FullName '需求.txt'
        if (-not (Test-Path $requirementFile)) {
            continue
        }

        $childRounds = Get-ThemeRounds -Dir $child.FullName -LogEntries $LogEntries
        $childRounds = Add-ImplementationRound -Dir $child.FullName -Rounds $childRounds -LogEntries $LogEntries

        if ($childRounds.Count -eq 0) {
            continue
        }

        $subUnits += [PSCustomObject]@{
            name            = $child.Name
            path            = $child.FullName
            rounds          = [PSCustomObject]@{
                total  = $childRounds.Count
                items  = $childRounds
                phases = @()
            }
            totalTime       = $null
            totalHumanChars = ($childRounds | Measure-Object -Property humanChars -Sum).Sum
            totalAiChars    = ($childRounds | Measure-Object -Property aiChars -Sum).Sum
        }
    }

    return $subUnits
}
```

分别调用：

```powershell
$resultFineTunings = Get-SubUnits -Dir $themePath -SubDirName '结果微调' -LogEntries $logEntries
$subThemes = Get-SubUnits -Dir $themePath -SubDirName '实施步骤' -LogEntries $logEntries
```

> 原则：通过参数复用同一函数处理两种子单元，避免复制粘贴两套逻辑。

#### 3.2.4 指标合并与 breakdown

主流程中：

```powershell
$allRounds = @($rounds) + @($resultFineTunings.rounds.items) + @($subThemes.rounds.items)
$totalHumanChars = ($allRounds | Measure-Object -Property humanChars -Sum).Sum
$totalAiChars = ($allRounds | Measure-Object -Property aiChars -Sum).Sum

# 总时间基于所有文件/日志的最早开始与最晚结束
$totalTime = Get-TotalTime -Dir $themePath -Rounds $allRounds -LogEntries $logEntries

# breakdown
$breakdown = [PSCustomObject]@{
    main = [PSCustomObject]@{
        rounds      = $rounds.Count
        timeMinutes = $mainTime.TotalMinutes
        humanChars  = ($rounds | Measure-Object -Property humanChars -Sum).Sum
        aiChars     = ($rounds | Measure-Object -Property aiChars -Sum).Sum
    }
    resultFineTunings = [PSCustomObject]@{
        rounds      = ($resultFineTunings | Measure-Object -Property { $_.rounds.total } -Sum).Sum
        timeMinutes = $rtTime.TotalMinutes
        humanChars  = ($resultFineTunings | ForEach-Object { $_.rounds.items } | Measure-Object -Property humanChars -Sum).Sum
        aiChars     = ($resultFineTunings | ForEach-Object { $_.rounds.items } | Measure-Object -Property aiChars -Sum).Sum
    }
    subThemes = [PSCustomObject]@{
        rounds      = ($subThemes | Measure-Object -Property { $_.rounds.total } -Sum).Sum
        timeMinutes = $stTime.TotalMinutes
        humanChars  = ($subThemes | ForEach-Object { $_.rounds.items } | Measure-Object -Property humanChars -Sum).Sum
        aiChars     = ($subThemes | ForEach-Object { $_.rounds.items } | Measure-Object -Property aiChars -Sum).Sum
    }
}
```

> 注意：`Get-TotalTime` / `Get-StageTime` 当前实现基于 `$Rounds` 参数计算起止时间，只需把合并后的 `$allRounds` 传入即可。但需要验证：当子单元轮次时间跨度与主轮次不连续时，`Get-ActiveDuration` 仍能正确计算活跃时长。

### 3.3 模板文件更新

#### 3.3.1 `summary_input_guide.md`

- 删除 `humanContentPreview` / `aiContentPreview` 说明。
- 新增「实施确认」轮次说明：
  - 当目录下存在 `实施文档.md` 时，会生成一个 `category = "实施确认"` 的轮次；
