# 结论：结果微调/02 最终设计

## 一、已确认事项

1. **「实施确认」作为普通轮次**：归入 `rounds.items`，通过 `category = "实施确认"` 标识，不再单独抽成字段。
2. **子主题 = `实施步骤/XX/`**：子主题目录位于 `实施步骤/` 下。
3. **结果微调 = `结果微调/XX/`**：结果微调目录独立分类。
4. **全量纳入指标**：根目录轮次 + 结果微调轮次 + 子主题轮次全部计入总时间、总字数、轮次分类、意图匹配率、纠偏率、失控率等指标。
5. **评价基于全量轮次**：`overview` 中的复杂度、协作顺畅度、摘要等评价必须基于全部轮次，不能只看根目录。
6. **增加 `breakdown` 字段**：JSON 是基础数据，越完整越好，因此增加各部分贡献拆分。

## 四、AI 生成 `Summary.json` 时的要求

基于以上输入结构，AI 在生成 `Summary.json` 时应：

1. 读取主轮次、结果微调、子主题的全部文件；
2. 对每个单元单独做轮次分类、阶段划分、纠偏/失控分析；
3. 主主题 `overview` 必须汇总全部内容；
4. `metrics` 按全量轮次计算；
5. 如需，可在 `overview.abstract` 或 `analysis` 中引用 `breakdown` 数据。
