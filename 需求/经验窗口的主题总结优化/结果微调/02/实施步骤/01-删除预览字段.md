# 步骤 01：删除预览字段

## 目标

从 `PrepareSummaryInput.ps1` 生成的 `summary_input.json` 中移除 `humanContentPreview` 和 `aiContentPreview` 字段，避免 AI 误将截断原文当作总结。

## 涉及文件

- `app/powershell/summary/PrepareSummaryInput.ps1`
- `app/templates/summary/summary_input_guide.md`

## 具体操作

### 1. 修改 `PrepareSummaryInput.ps1`

在 `Get-ThemeRounds` 函数中，找到构造 round 对象的代码块：

```powershell
$rounds += [PSCustomObject]@{
    roundIndex          = $ver - 1
    stage               = '讨论'
    humanFile           = $humanFileName
    aiFile              = $aiFile.Name
    startTime           = $humanTime.ToString('yyyy-MM-dd HH:mm:ss')
    endTime             = $aiTime.ToString('yyyy-MM-dd HH:mm:ss')
    durationMinutes     = $duration
    humanChars          = Get-CharCount -Path (Join-Path $Dir $humanFileName)
    aiChars             = Get-CharCount -Path $aiFile.FullName
    humanContentPreview = Get-TextPreview -Path (Join-Path $Dir $humanFileName)
    aiContentPreview    = Get-TextPreview -Path $aiFile.FullName
}
```

删除 `humanContentPreview` 和 `aiContentPreview` 两行：

```powershell
$rounds += [PSCustomObject]@{
    roundIndex      = $ver - 1
    stage           = '讨论'
    humanFile       = $humanFileName
    aiFile          = $aiFile.Name
    startTime       = $humanTime.ToString('yyyy-MM-dd HH:mm:ss')
    endTime         = $aiTime.ToString('yyyy-MM-dd HH:mm:ss')
    durationMinutes = $duration
    humanChars      = Get-CharCount -Path (Join-Path $Dir $humanFileName)
    aiChars         = Get-CharCount -Path $aiFile.FullName
}
```

`Get-TextPreview` 函数本身可以保留（不调用即可），避免顺手删除无关代码。

### 2. 修改 `summary_input_guide.md`

删除或改写以下说明：

> `humanContentPreview` / `aiContentPreview`：内容前约 200 字预览，供分类判断使用。  
> 注意：`humanContentPreview` 和 `aiContentPreview` 是截断预览。如需完整内容，请按 `path` 读取原始文件。

替换为：

> `humanFile` / `aiFile`：每轮对应的人输入文件和 AI 输出文件的相对路径。  
> 如需理解每轮具体内容，请直接按这些路径读取原始文件。`summary_input.json` 中不再提供内容预览。

## 验证

在「周报时间」主题上运行 `PrepareSummaryInput.ps1`，检查生成的 `summary_input.json`：

- `rounds.items` 中不应再包含 `humanContentPreview` / `aiContentPreview`。
- `summary_input_guide.md` 中不应再提及这两个字段。
