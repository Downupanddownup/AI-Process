# 步骤 05：更新模板规则文件

## 目标

同步更新 `summary_input_guide.md`、`summary_template.json`、`summary_rules.md`，使 AI 能正确理解新的输入结构。

## 涉及文件

- `app/templates/summary/summary_input_guide.md`
- `app/templates/summary/summary_template.json`
- `app/templates/summary/summary_rules.md`

## 具体操作

### 1. 更新 `summary_input_guide.md`

#### 删除预览字段说明

删除或改写关于 `humanContentPreview` / `aiContentPreview` 的段落，改为：

> `humanFile` / `aiFile`：每轮对应的人输入文件和 AI 输出文件的相对路径。如需理解每轮具体内容，请直接按这些路径读取原始文件。

#### 新增「实施确认」说明

在轮次字段说明中增加：

> 当目录下存在 `实施文档.md` 时，会在 `rounds.items` 末尾追加一个 `category = "实施确认"` 的轮次。该轮次的 `humanFile` 为上一个轮次的人文件，`aiFile` 为 `实施文档.md`，代表需求收敛、开始执行的转折点。

#### 新增 `resultFineTunings` 说明

> `resultFineTunings`：结果微调子议题数组。每个元素结构与主主题 `rounds` 一致，对应 `结果微调/XX/` 子目录下的沟通轮次。

#### 新增 `subThemes` 说明

> `subThemes`：子主题数组。每个元素结构与主主题 `rounds` 一致，对应 `实施步骤/XX/` 子目录下的沟通轮次。

#### 新增 `breakdown` 说明

> `breakdown`：拆分了主讨论、结果微调、子主题三部分的轮次数、时长、字数贡献，便于后续展示。

```json
  "main": { "rounds": 0, "timeMinutes": 0, "humanChars": 0, "aiChars": 0 },
  "resultFineTunings": { "rounds": 0, "timeMinutes": 0, "humanChars": 0, "aiChars": 0 },
  "subThemes": { "rounds": 0, "timeMinutes": 0, "humanChars": 0, "aiChars": 0 }
}
```

### 3. 更新 `summary_rules.md`

#### 增加「实施确认」分类

在轮次分类章节中增加：

```markdown
### 实施确认

- 判定：该轮次 `aiFile` 为 `实施文档.md`。
- 含义：需求收敛、开始执行的转折点，标志着从讨论阶段进入实施阶段。
- 注意：它不属于 6 类基础轮次分类（正常推进、细化完善、探索发散、纠偏拉回、极端失控、确认空转），而是独立的里程碑节点。
```

#### 说明主主题分析应综合全部内容

在分析规则章节中增加：

```markdown
### 全量分析原则

- `overview` 中的摘要、复杂度、协作顺畅度必须基于主轮次 + `resultFineTunings` + `subThemes` 的全部轮次综合判断。
- `metrics` 中的轮次分类统计、意图匹配率、纠偏率、失控率应按全量轮次计算。
- 结果微调和子主题中的纠偏/失控情况，应在 `analysis` 中汇总体现。
```

## 验证

检查三份模板文件：

- `summary_input_guide.md` 不再提及 `humanContentPreview` / `aiContentPreview`。
- `summary_template.json` 包含 `resultFineTunings`、`subThemes`、`breakdown`。
- `summary_rules.md` 包含「实施确认」和「全量分析原则」说明。
