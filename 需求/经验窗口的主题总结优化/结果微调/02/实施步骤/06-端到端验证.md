# 步骤 06：端到端验证

## 目标

在「周报时间」主题上验证新流程：从 `summary_input.json` 生成，到 Agent 生成 `Summary.json`，再到 Markdown 渲染，确保结构正确、无报错。

## 验证步骤

### 1. 生成本地输入文件

在 PowerShell 中运行：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\NotSystem\ClaudeCodeTool\AIProcess\app\powershell\summary\PrepareSummaryInput.ps1" `
  -ThemePath "C:\NotSystem\ClaudeCodeTool\AIProcess\需求\周报时间" `
  -OutputFile "C:\NotSystem\ClaudeCodeTool\AIProcess\需求\周报时间\.aiprocess\_tmp\summary_input_test.json"
```

检查生成的 `summary_input_test.json`：

- [ ] 不含 `humanContentPreview` / `aiContentPreview`。
- [ ] `rounds.items` 最后一轮为 `category = "实施确认"`。
- [ ] `resultFineTunings` 非空，每个元素含完整 `rounds.items`。
- [ ] `subThemes` 非空（如果存在实施步骤），每个元素含完整 `rounds.items`。
- [ ] `breakdown` 存在，且三部分数值相加等于总量。
- [ ] `totalHumanChars` / `totalAiChars` 包含全量。

### 2. 触发 Agent 生成 Summary.json

在 AI Process 中打开「经验总结」窗口，选择「周报时间」主题，点击「生成总结」。

观察：

- [ ] Agent 是否成功读取新的 `summary_input.json`。
- [ ] `.aiprocess\Summary.json` 是否生成。
- [ ] `Summary.json` 中是否包含 `resultFineTunings` / `subThemes` / `breakdown` 字段（如模板要求）。

### 3. 查看 Markdown

点击「查看总结」，观察：

- [ ] `Summary.md` 是否正常生成。
- [ ] 内容是否覆盖主轮次、结果微调、子主题。

如果 Markdown 渲染因新字段缺失而报错，记录错误但不修改 `ConvertSummaryToMarkdown.ps1`（本次范围外）。

### 4. 回归测试

在另一个无子目录的主题（例如「闭环」）上重复验证：

- [ ] 无 `结果微调/` 和 `实施步骤/` 时，`resultFineTunings` 和 `subThemes` 为空数组。
- [ ] `breakdown` 中对应部分为 0。
- [ ] 整体流程不报错。

## 通过标准

- `PrepareSummaryInput.ps1` 在「周报时间」上生成的 `summary_input.json` 结构符合预期。
- Agent 能基于新结构成功生成 `Summary.json`。
- 「闭环」等无子目录主题仍能正常工作。

## 失败处理

如验证失败：

1. 查看 `summary_input_err.txt` 或 `Summary.json` 是否生成。
2. 回退到步骤 01–05 逐一排查。
3. 必要时还原 `PrepareSummaryInput.ps1` 备份。
