# 回复确认：子主题识别规则与处理方案

## 一、关键澄清确认

1. **「实时确认」时间计算**：OK，按 last reply → 实施文档.md 计算耗时。
2. **「资助题」是语音输入错误**：正确说法是 **「子主题」**，即父主题的子级主题。
3. **子主题识别特征**：子目录下存在 `需求.txt` 文件。

## 二、子主题扫描规则建议

基于你的描述，我建议采用一个**通用且稳定的识别规则**：

> 在主题目录下，递归扫描所有一级子目录（排除 `.aiprocess`、`.git`、`.idea` 等系统目录），只要某个子目录下存在 `需求.txt`，就把它识别为一个子主题。

这个规则的好处：

- 不限定必须是 `结果微调/XX/` 或 `实施步骤/XX/`，只要结构符合即可；
- 以后如果有新的子议题类型（例如 `历史主题/XX/`、`问题复盘/XX/`），无需改代码；
- 以 `需求.txt` 作为判断依据，符合项目「一轮 = 人 .txt → AI .md」的基本约定。

## 三、子主题处理逻辑

### 1. 在 `PrepareSummaryInput.ps1` 中

新增子主题扫描步骤：

```powershell
function Get-SubThemes {
    param([string]$Dir, [array]$LogEntries)

    $subThemes = @()
    $items = Get-ChildItem -LiteralPath $Dir -Directory -Force | Where-Object {
        $excludedDirs -notcontains $_.Name
    }

    foreach ($item in $items) {
        $requirementFile = Join-Path $item.FullName '需求.txt'
        if (Test-Path -Path $requirementFile) {
            $subRounds = Get-ThemeRounds -Dir $item.FullName -LogEntries $LogEntries
            # 同样检测实施文档.md
            $subRounds = Add-ImplementationRound -Dir $item.FullName -Rounds $subRounds -LogEntries $LogEntries

            $subThemes += [PSCustomObject]@{
                themeName       = $item.Name
                themePath       = $item.FullName
                rounds          = $subRounds
                totalTime       = ...
                totalHumanChars = ...
                totalAiChars    = ...
            }
        }
    }

    return $subThemes
}
```

### 2. 子主题是否参与主主题分析

你的原话是「结果微调里面的那个轮次也得放进来一起讨论」。我理解这个「一起讨论」有两层含义：

- **结构层**：子主题仍然作为 `subThemes` 独立存在，不散落到主主题 `rounds.items` 里；
- **分析层**：AI 在生成主主题 `Summary.json` 的 `overview`、`analysis`、`metrics` 时，应该把子主题的内容纳入考量。

例如：

- `overview.abstract` 中应提到「本主题包含 N 个结果微调子议题」；
- `analysis` 中如果子主题出现纠偏/失控，应在汇总时一并说明；
- `metrics` 中可以单独标注子主题数量，但核心指标仍以主主题轮次为准。

如果你希望更彻底——把子主题轮次**扁平合并**到主主题 `rounds.items` 里——也可以，但这会让 `roundIndex` 跨越不同子目录，timeline 和 phases 会变得混乱。所以我建议保留 `subThemes` 结构，只在分析层面「一起讨论」。
