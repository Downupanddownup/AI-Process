# 回复确认与具体修改方案

## 一、关于 `humanContentPreview` / `aiContentPreview`

你的判断是对的：直接删掉。

原因：

- 脚本截断的预览会让 AI 误以为这就是完整信息，从而不再读原文件；
- 一旦 AI 只依赖截断内容，生成的 `Summary.json` 必然失真；
- 明确让 AI 读原文件是最稳的做法。

### 需要修改的文件

1. **`PrepareSummaryInput.ps1`**
   - 删除 `Get-TextPreview` 函数的调用（可保留函数本身，以后可能复用）；
   - 在 `Get-ThemeRounds` 构建 round 对象时，不再生成 `humanContentPreview` / `aiContentPreview`。

2. **`summary_input_guide.md`**
   - 删除或改写关于 `humanContentPreview` / `aiContentPreview` 的说明；
   - 明确告诉 AI：`summary_input.json` 只提供结构、时间、字数、文件路径；每轮具体内容请按 `humanFile` / `aiFile` 路径读取原文件。

3. **`summary_template.json`**
   - 当前 `rounds.items` 里已经没有 `humanContentPreview` / `aiContentPreview`，只有 `humanSummary` / `aiSummary`；
   - 这一步**不需要改**。

## 二、关于 `实施文档.md` 作为特殊轮次「实时确认」

### 你的逻辑整理

- `实施文档.md` 不是独立产生的，它紧跟最后一个用户回复文件；
- 这个特殊轮次的分类名称为：**实时确认**；
- 文件界定：
  - `humanFile` = 最后一个用户回复文件（例如 `对v4的回复.txt`）
  - `aiFile` = `实施文档.md`

### 处理逻辑建议

在 `PrepareSummaryInput.ps1` 的 `Get-ThemeRounds` 之后，追加一段特殊轮次检测：

```powershell
# 示例逻辑
$implDoc = Join-Path $Dir '实施文档.md'
if (Test-Path $implDoc) {
    $lastRound = $rounds | Sort-Object roundIndex | Select-Object -Last 1
    $lastHumanFile = ""
    if ($lastRound) {
        $lastHumanFile = $lastRound.humanFile
    }

    $implTime = Get-LogTimeForFile -LogEntries $LogEntries -FileName '实施文档.md' -FallbackTime (Get-Item $implDoc).CreationTime

    $rounds += [PSCustomObject]@{
        roundIndex      = if ($lastRound) { $lastRound.roundIndex + 1 } else { 0 }
        stage           = '实时确认'
        humanFile       = $lastHumanFile
        aiFile          = '实施文档.md'
        startTime       = $implTime.ToString('yyyy-MM-dd HH:mm:ss')
        endTime         = $implTime.ToString('yyyy-MM-dd HH:mm:ss')
        durationMinutes = 1
        humanChars      = 0
        aiChars         = Get-CharCount -Path $implDoc
        category        = '实时确认'
        isExtreme       = $false
        extremeReason   = ''
    }
}
```

注意：因为当前 `Get-ThemeRounds` 返回的对象里还没有 `category` / `isExtreme` / `extremeReason` 字段（这些是让 AI 在 Summary.json 里填的），所以 `PrepareSummaryInput.ps1` 里可以不填，只把 `stage` 标记为 `实时确认` 即可。

### 需要修改的文件

1. **`PrepareSummaryInput.ps1`**
   - 在 `Get-ThemeRounds` 末尾或主流程中增加「实施文档特殊轮次」生成逻辑。

2. **`summary_input_guide.md`**
   - 说明 `rounds` 中会出现 `stage = "实时确认"` 的特殊轮次；
   - 说明该轮次的 `humanFile` 是最后一个用户回复文件，`aiFile` 是 `实施文档.md`；
   - 说明该轮次代表需求收敛、开始执行的转折点。

3. **`summary_rules.md`**
   - 在轮次分类说明中增加「实时确认」这一特殊类型；
   - 说明它不属于 6 类基础分类，而是里程碑节点；
   - 在阶段划分或分析规则中说明：该轮次可作为「讨论 → 执行」转折点的标志。

4. **`summary_template.json`**
   - 当前 `category` 字段是自由字符串，可以容纳 `"实时确认"`；
   - 如果希望限制可选值，可以在 guide 或 rules 里说明，**模板本身不一定要改**。
