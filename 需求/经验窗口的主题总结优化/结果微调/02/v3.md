# 回复确认：「实施确认」与「结果微调」遗漏

## 一、「实施确认」分类名称与时间计算确认

分类名称确定为：**实施确认**。

时间计算逻辑如下：

- `humanFile` = 最后一个用户回复文件（例如 `对v4的回复.txt`）
- `aiFile` = `实施文档.md`
- `startTime` = 最后一个用户回复文件的时间
- `endTime` = `实施文档.md` 的创建时间
- `durationMinutes` = `endTime - startTime`，若小于 1 则取 1

这样「实施确认」就有真实耗时，而不是占位 1 分钟。

## 二、关于「结果微调」遗漏

### 问题确认

我检查了 `PrepareSummaryInput.ps1`，发现问题确实存在。

当前 `Get-ThemeRounds` 函数只扫描主题根目录下的文件：

```powershell
$files = Get-ChildItem -LiteralPath $Dir -File -Force | Where-Object { ... }
```

它**没有递归扫描 `结果微调/XX/` 子目录**。所以像「周报时间」主题下的 `结果微调/01/`、`结果微调/02/` 都被忽略了。

我同时也看了生成的 `Summary.json`（周报时间），确实只有 4 个根目录轮次，既没有「实施确认」轮次，也没有 `subThemes`。

### 处理建议

`结果微调/XX/` 下的每个子目录都是一个独立的子议题，应按子主题处理。`summary_template.json` 中已经预留了 `subThemes: []` 字段，正好可以利用。

#### `subThemes` 结构建议

```json
"subThemes": [
  {
    "themeName": "01",
    "themePath": "C:\\...\\周报时间\\结果微调\\01",
    "totalRounds": 3,
    "rounds": [ ... ],
    "overview": { ... },
    "analysis": { ... }
  }
]
```

#### 实现思路

1. 在 `PrepareSummaryInput.ps1` 主流程中，扫描主题目录下的 `结果微调/` 子目录。
2. 对每个 `XX/` 子目录，递归调用类似 `Get-ThemeRounds` 的逻辑，生成该子主题的轮次。
3. 对每个子主题，同样检测是否存在 `实施文档.md`，生成「实施确认」轮次。
4. 将所有子主题信息写入 `summary_input.json` 的 `subThemes` 字段。

#### 关于 AI 的处理

AI 在生成 `Summary.json` 时：

- 主主题总结仍基于根目录轮次；
- 子主题作为补充信息，在 `subThemes` 中各自生成总结；
- `overview` 中可适当提及存在哪些子议题（例如「包含 2 个结果微调子议题」）。
