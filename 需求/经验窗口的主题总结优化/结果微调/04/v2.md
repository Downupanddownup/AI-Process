# 方案 B 完整指令

## 一次性强制提交

在 AIProcess 项目根目录下执行：

```bash
cd C:\NotSystem\ClaudeCodeTool\AIProcess

# 1. 强制添加被忽略的目录
git add -f "需求/经验窗口的主题总结优化/"

# 2. 查看暂存区状态，确认文件已加入
git status

# 3. 提交（修改提交信息为你想要的）
git commit -m "docs: 提交经验窗口的主题总结优化沟通记录"

# 4. 推送到 GitHub
git push
```

## 注意事项

- 由于 `.gitignore` 仍然忽略 `需求/`，以后该主题下**新增或修改的文件**仍然不会自动被 Git 追踪。
- 每次有更新后，都需要再次执行 `git add -f 需求/经验窗口的主题总结优化/`。
- 如果后续想一次性解决，建议还是改用方案 A（修改 `.gitignore` 加例外规则）。

## 如果想确认哪些文件会被提交

在执行 `git add -f` 之前，可以先看：

```bash
git add -f -n "需求/经验窗口的主题总结优化/"
```

`-n` 是 dry-run，只显示会添加哪些文件，不会真正加入暂存区。
