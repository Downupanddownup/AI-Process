# 关于单独提交「经验窗口的主题总结优化」到 GitHub

## 问题

当前 `.gitignore`  likely 把整个 `需求/` 目录都忽略了，但现在希望把 `需求/经验窗口的主题总结优化/` 这个子目录单独提交到 GitHub。

## 原因

Git 的忽略规则是：如果父目录 `需求/` 整个被忽略，Git 不会进入该目录去检查子目录的例外规则。所以单纯在 `.gitignore` 里写 `!需求/经验窗口的主题总结优化/` 是不生效的。

## 解决方案

### 方案 A（推荐）：修改 `.gitignore` 加例外规则

把 `.gitignore` 中关于 `需求/` 的规则从：

```gitignore
需求/
```

改成：

```gitignore
# 忽略需求目录下所有内容
需求/*

# 但不忽略「经验窗口的主题总结优化」这个主题
!需求/经验窗口的主题总结优化/
!需求/经验窗口的主题总结优化/**
```

注意：

- `需求/*` 只忽略目录里的内容，不忽略 `需求/` 目录本身，这样 Git 才会继续往里扫描例外规则。
- `!需求/经验窗口的主题总结优化/` 取消忽略该目录。
- `!需求/经验窗口的主题总结优化/**` 取消忽略该目录下所有子文件和子目录。

修改后执行：

```bash
git add 需求/经验窗口的主题总结优化/
git status
```

如果看到文件被加入暂存区，就说明规则生效了。

### 方案 B：强制添加（简单但不持久）

不想改 `.gitignore` 的话，可以强制 add：

```bash
git add -f 需求/经验窗口的主题总结优化/
```

但这只是本次生效，以后该目录下新增或修改的文件仍然会被 `.gitignore` 忽略，需要每次都 `git add -f`。

## 建议

用**方案 A**，一次性解决问题，后续该主题下的沟通文件、实施文档等都能正常提交。
