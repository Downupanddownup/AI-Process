# 回复确认：去掉 stageTime，文件清单使用相对路径

## 一、关于 `stageTime`

同意去掉。

`discussion / execution / tweak` 这三个阶段概念确实不好定义，而且在当前结构下容易和 `rounds`、`resultFineTunings`、`subThemes` 产生混淆。直接移除更干净。

## 二、保留的时间字段

去掉 `stageTime` 后，时间相关字段保留以下这些：

| 字段 | 含义 |
|---|---|
| `totalTime` | 整个主题（根目录 + 结果微调 + 子主题）的活跃总时长 |
| `breakdown.main.timeMinutes` | 根目录沟通时长 |
| `breakdown.resultFineTunings.timeMinutes` | 所有结果微调单元时长之和 |
| `breakdown.subThemes.timeMinutes` | 所有子主题单元时长之和 |
| `resultFineTunings[i].totalTime` | 单个结果微调单元的时长 |
| `subThemes[i].totalTime` | 单个子主题单元的时长 |

这些信息足够完整，也避免了阶段概念的模糊性。

## 三、文件清单路径问题

你说得对，当前 `files` 里的 `path` 只给了文件名，根本找不到文件在哪。必须改成**从主题根目录出发的相对路径**。

### 修正后的 `files` 结构

```json
"files": {
  "humanFiles": [
    { "path": "需求.txt", "createdAt": "...", "charCount": 180 },
    { "path": "对v1的回复.txt", "createdAt": "...", "charCount": 97 },
    ...
  ],
  "aiFiles": [
    { "path": "v1.md", "createdAt": "...", "charCount": 2270 },
    ...
  ],
  "resultFineTunings": {
    "01": {
      "humanFiles": [
        { "path": "结果微调/01/需求.txt", "createdAt": "...", "charCount": 254 },
        { "path": "结果微调/01/对v1的回复.txt", "createdAt": "...", "charCount": ... },
        ...
      ],
      "aiFiles": [
        { "path": "结果微调/01/v1.md", "createdAt": "...", "charCount": 2085 },
        ...
      ]
    },
    "02": { ... }
  },
  "subThemes": {
    "01-历史目录": {
      "humanFiles": [
        { "path": "实施步骤/01-历史目录/需求.txt", ... },
        ...
      ],
      "aiFiles": [
        { "path": "实施步骤/01-历史目录/v1.md", ... },
        ...
      ]
    }
  }
}
```

### 路径规则

- 根目录文件：`需求.txt`、`v1.md` 等；
- 结果微调文件：`结果微调/01/需求.txt`、`结果微调/01/v1.md` 等；
- 子主题文件：`实施步骤/01-历史目录/需求.txt`、`实施步骤/01-历史目录/v1.md` 等。

这样 AI 看到路径就能直接定位文件。
