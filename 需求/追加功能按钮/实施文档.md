# 实施文档

## 1. 项目目标

在现有 AIProcess 工具基础上，去掉【先别改】按钮，新增【复执行】按钮及其配套的【步】复选框，以支持从 `实施文档.md` 进入正式执行阶段的快捷触发。

## 2. 变更范围

### 2.1 去掉的内容

- 【先别改】按钮及其对应的 `CopyDiscussPrompt` 函数。

### 2.2 新增的内容

- 【复执行】按钮：复制执行阶段提示词到剪贴板。
- 【步】复选框：控制执行模式（直接执行 / 步骤拆解模式）。
- 模板文件 `execute_prompt.txt`：直接执行模式的提示词模板。
- 模板文件 `execute_step_prompt.txt`：步骤拆解模式的提示词模板。

### 2.3 调整的内容

- 界面布局：第 5 行由【复关系 + 先别改】调整为【复关系 + 复执行 + 步】。
- 控件可用性控制：`SetControlsEnabled` 中移除【先别改】引用，增加【复执行】和【步】。
- 默认文件初始化：`EnsureDefaultFiles` 中增加两个新模板的自动创建逻辑。

## 3. 界面结构

### 3.1 整体布局

窗口尺寸保持现有默认值（宽 210px × 高 182px），行数不变。

| 行 | 控件 | 说明 |
|---|---|---|
| 1 | 当前目录显示 + 设目录 | 状态与目录设置 |
| 2 | 用IDEA开 + 禁改正式 | 复选框开关 |
| 3 | 建需求 + 复需求 | 需求阶段动作 |
| 4 | 建回复 + 复回复 + 实 | 回复阶段动作 |
| 5 | 复关系 + 复执行 + 步 | **本次变更区域** |

### 3.2 【复执行】+【步】布局细则

- 【复执行】按钮：宽 60px，高 24px，与现有按钮样式一致。
- 【步】复选框：紧跟【复执行】右侧，间距 6px，垂直偏移 +4px（与【实】复选框的对齐方式一致）。
- 【步】复选框文字为单个字符"步"，默认状态为**不勾选**。
- 【步】复选框状态**不持久化**到 `settings.ini`，每次启动默认不勾选。

## 4. 功能详细说明

### 4.1 【复执行】按钮

#### 执行条件

1. 已设置当前目录。
2. 当前目录下存在 `实施文档.md`。

#### 执行逻辑

1. 校验 `CurrentDir` 是否已设置，未设置则提示"请先设置当前主题目录"。
2. 拼接实施文档路径：`当前目录\实施文档.md`。
3. 检查该文件是否存在，不存在则提示"当前目录下未找到 实施文档.md"并终止。
4. 根据【步】复选框状态，加载对应模板：
   - 【步】**未勾选**：加载 `execute_prompt.txt`。
   - 【步】**已勾选**：加载 `execute_step_prompt.txt`。
5. 替换模板中的 `{{filePath}}` 为实施文档的完整路径。
6. 将结果复制到剪贴板。
7. 显示反馈：
   - 不勾选【步】时："执行提示词已复制"
   - 勾选【步】时："步骤模式提示词已复制"

#### 特殊规则

- **不受"禁改正式"影响**：即使"禁改正式"复选框处于勾选状态，【复执行】复制的提示词也不追加 `no_modify_prompt.txt` 的内容。
- 提示词中**不补充**当前目录上下文（如文件关系说明），保持极简。

### 4.2 【步】复选框

#### 行为说明

- 仅作为【复执行】按钮的附属状态开关，自身不触发任何动作。
- 状态不持久化，每次启动工具默认为不勾选。
- 可用性随目录设置状态联动：未设置目录时，【步】与【复执行】一同置灰。

## 5. 提示词模板

### 5.1 execute_prompt.txt（直接执行模式）

```
请你根据当前的实施文档：{{filePath}}，修改正式代码和文件，完成整个方案的落地。
```

### 5.2 execute_step_prompt.txt（步骤拆解模式）

```
请你查看当前的实施文档：{{filePath}}。

请先判断该方案的内容复杂度，是否需要拆解为多个实施步骤：
- 如果不需要拆解，请直接修改正式代码和文件，完成整个方案的落地。
- 如果需要拆解，请在当前目录下创建【实施步骤】目录，按照合理的执行顺序为每个步骤创建 md 文件（格式如 01-xxx.md、02-xxx.md 等），确保所有步骤合起来覆盖实施文档的全部内容。步骤文档写完后请先停下来，告诉我已经写好，等待我验收。不要在未获得我确认前直接修改正式代码。
```

### 5.3 模板存放位置

`app/templates/execute_prompt.txt`  
`app/templates/execute_step_prompt.txt`

## 6. 技术实现要点

### 6.1 全局变量调整

在 `app/AIProcess.ahk` 中：

- **移除**：`global CopyDiscussButton := ""
- **新增**：`global CopyExecuteButton := ""`  
- **新增**：`global ExecuteStepCheckbox := ""`

### 6.2 界面创建调整

在 `CreateMainGui()` 中：

1. 移除【先别改】按钮的创建代码。
2. 在【复关系】右侧新增【复执行】按钮（使用相同的 `actionButtonWidth` / `actionButtonHeight` / `actionGap` 变量）。
3. 在【复执行】右侧新增【步】复选框（使用 `x+6 yp+4` 对齐，与【实】复选框一致）。
4. 【步】复选框不绑定 `OnEvent`，因为它只是状态存储。

### 6.3 控件可用性调整

在 `SetControlsEnabled(enabled)` 中：

1. 移除 `CopyDiscussButton` 的启用/禁用控制。
2. 增加 `CopyExecuteButton` 和 `ExecuteStepCheckbox` 的启用/禁用控制。

### 6.4 新增复制函数

新增 `CopyExecutePrompt(*)` 函数：

1. 调用 `EnsureCurrentDirectory()` 校验目录。
2. 拼接实施文档路径并检查文件存在性。
3. 根据 `ExecuteStepCheckbox.Value` 决定加载哪个模板。
4. 执行 `StrReplace` 替换 `{{filePath}}`。
5. 写入剪贴板并显示差异化反馈。

**注意**：此函数不调用 `AppendNoModifyPromptIfNeeded`。

### 6.5 默认文件初始化调整

在 `EnsureDefaultFiles()` 中：

增加两个 `if !FileExist` 分支，分别生成 `execute_prompt.txt` 和 `execute_step_prompt.txt` 的默认内容（见第 5 节）。

## 7. 按钮可用性规则

### 7.1 未设置目录时

除"设目录"外，所有功能不可用。包含【复执行】和【步】。

### 7.2 已设置目录但缺少实施文档.md 时

【复执行】按钮**仍然保持可用**（UI 层面不禁用），但点击后内部校验提示"当前目录下未找到 实施文档.md"。  
（与现有"建回复"逻辑一致：有目录但无 vX.md 时，按钮可用，点击后报错。）

## 8. 验收标准

满足以下条件即视为本次变更完成：

1. 【先别改】按钮已从界面和代码中完全移除。
2. 界面上出现【复执行】按钮和【步】复选框，布局与【复回复】+【实】一致。
3. 点击【复执行】时：
   - 未设置目录 → 提示"请先设置当前主题目录"。
   - 已设置目录但无 `实施文档.md` → 提示"当前目录下未找到 实施文档.md"。
   - 已设置目录且有 `实施文档.md`，不勾选【步】 → 复制直接执行提示词，反馈"执行提示词已复制"。
   - 已设置目录且有 `实施文档.md`，勾选【步】 → 复制步骤拆解提示词，反馈"步骤模式提示词已复制"。
4. 【复执行】复制的提示词不受"禁改正式"复选框影响。
5. 【步】复选框状态不持久化，每次启动默认不勾选。
6. `app/templates/` 目录下自动生成 `execute_prompt.txt` 和 `execute_step_prompt.txt`（缺失时）。
7. 现有其他按钮功能不受影响。

## 9. 结论

本次变更的核心是在现有沟通流程（需求 → 回复 → 实施文档）的基础上，补齐最后一个动作：**从实施文档触发正式执行**。通过【复执行】+【步】的组合，用户可以一键复制直达执行阶段的提示词，并在"直接动手"与"先拆步骤"两种模式间快速切换。

变更量较小，主要集中在 UI 替换、新增复制函数、新增两个模板文件，不涉及现有核心流程（目录管理、版本扫描、回复流程等）的改动。
