## 2. 第一版产品定位

第一版产品定位为：

`一个运行在 Windows 上的轻量级 AI 协作快捷面板`

它的主要价值不是复杂自动化，而是减少你在 AI 协作过程中的重复操作。

第一版只解决以下高频问题：

1. 设置当前主题目录
2. 快速创建固定文件
3. 快速复制固定提示词
4. 快速说明当前目录下文档之间的关系
5. 通过快捷键快速呼出窗口
6. 通过托盘常驻，避免频繁开关程序

## 4. 最终技术选型

### 4.1 主技术路线

第一版推荐技术方案：

`Tauri + Vanilla TypeScript + HTML/CSS + JSON 配置文件`


### 4.2 选择原因

#### 选择 Tauri 的原因

1. 比 Electron 更轻
2. 内存占用更低
3. 启动速度更快
4. 支持桌面窗口控制
5. 支持系统托盘
6. 支持全局快捷键
7. 支持剪贴板
8. 支持本地文件操作
9. 适合做常驻型小工具


#### 选择 Vanilla TypeScript 的原因

1. 第一版界面简单
2. 不需要复杂组件体系
3. 不需要复杂前端状态管理
4. 工程文件会更少
5. 更利于保持程序轻量


#### 选择 JSON 配置文件的原因

1. 提示词可集中维护
2. UI 文案可集中维护
3. 修改后重启即可生效
4. 不需要改动业务代码就能调整大部分文字内容


## 5. 总体工程原则

开发时必须坚持以下原则：

1. 提示词正文不能散落在代码各处
2. UI 文案不能散落在组件逻辑里
3. 文件规则统一放到文件服务模块
4. 窗口行为统一放到窗口管理模块
5. 单个按钮逻辑不能写成长篇混合代码
6. 所有高频可改内容优先配置化
7. 第一版优先轻量和稳定，不追求扩展过度


## 6. 最终行为规则

### 6.1 程序启动行为

1. 用户双击程序后，程序启动
2. 程序创建托盘图标
3. 主窗口默认显示
4. 主窗口默认置顶
5. 主窗口出现在屏幕顶部区域


### 6.2 程序关闭行为

1. 点击右上角关闭按钮时，不真正退出
2. 点击关闭按钮后，窗口隐藏到托盘
3. 真正退出程序只能通过托盘菜单 `退出`


### 6.3 程序最小化行为

1. 点击最小化按钮时，窗口隐藏到托盘
2. 不在任务栏保留普通窗口占位


### 6.4 托盘行为

托盘菜单第一版只保留两个选项：

1. `显示`
2. `退出`

双击托盘图标行为：

1. 恢复显示主窗口
2. 将窗口激活到最前面
3. 保持置顶状态


### 6.5 快捷键行为

第一版默认快捷键：

`F2`

行为规则：

1. 窗口隐藏时，按 `F2` 呼出窗口
2. 窗口显示时，按 `F2` 激活窗口并拉到最前面

第一版实现方式：

1. 快捷键写在配置文件中
2. 用户后续可以通过修改配置文件调整
3. 第一版暂不提供界面内快捷键设置功能


### 6.6 置顶与拖拽行为

1. 主窗口显示时始终置顶
2. 窗口支持拖拽移动
3. 第一版采用“顶部标题区域拖拽”
4. 由于窗口较小，置顶不会构成明显遮挡


## 7. 固定文件规则

### 7.1 固定文件名

1. `需求.txt`
2. `vX.md`
3. `对vX的回复.txt`
4. `实施文档.md`


### 7.2 版本识别规则

1. 扫描当前目录下所有符合 `v数字.md` 的文件
2. 取编号最大的 `vX.md` 作为当前最新版本
3. 对应回复文件名为 `对vX的回复.txt`

示例：

1. 如果目录中有 `v1.md`、`v2.md`、`v5.md`
2. 当前最新版本视为 `v5.md`
3. 回复文件视为 `对v5的回复.txt`


### 7.3 文件已存在时的规则

#### 创建 `需求.txt`

1. 若文件不存在，则创建
2. 若文件已存在，则提示 `文件已存在`
3. 若开关开启，则继续用 IDEA 打开该文件


#### 创建 `对vX的回复.txt`

1. 若文件不存在，则创建
2. 若文件已存在，则提示 `文件已存在`
3. 若开关开启，则继续用 IDEA 打开该文件


## 8. 提示词规则

### 8.1 需求提示词

复制内容必须包含：

1. `需求.txt` 的完整路径
2. 指示 AI 查看该文件
3. 如果有新的想法或问题，则创建 `v1.md`
4. 如果没有新的问题，则创建 `实施文档.md`


### 8.2 回复提示词

复制内容必须包含：

1. 最新版本对应的 `对vX的回复.txt` 完整路径
2. 指示 AI 查看该回复文件
3. 如果仍有新问题，则继续创建下一版 `vX.md`
4. 如果没有新问题，则创建 `实施文档.md`


### 8.3 文件关系说明

复制内容必须说明：

1. `需求.txt` 是原始需求说明
2. `vX.md` 是 AI 沟通过程中的版本文档
3. `对vX的回复.txt` 是用户对对应版本的回复
4. `实施文档.md` 是整个需求沟通收敛后的最终结论文件，也是后续正式实施时最重要的执行依据
5. 这些文件按时间顺序构成完整沟通过程
6. 新会话接管时，应优先阅读这些文件完成上下文重建


## 9. 配置驱动设计

### 9.1 设计目标

配置层需要承担以下职责：

1. 集中管理提示词内容
2. 集中管理按钮文案
3. 集中管理错误提示
4. 集中管理快捷键配置
5. 支持修改后重启程序生效


### 9.2 配置目录建议

```text
config/
  prompts.json
  ui.json
  app.json
```


### 9.3 `prompts.json`

职责：

1. 存放需求提示词
2. 存放回复提示词
3. 存放文件关系说明文案

建议结构：

```json
{
  "requirementPrompt": "这是一个需求文件，请你查看这个文件：{{filePath}}。如果你有新的想法或问题，请创建下一版文档；如果没有新的问题，请创建实施文档.md。",
  "replyPrompt": "这是一个回复文件，请你查看这个文件：{{filePath}}。如果你还有新的想法或问题，请继续创建下一版 vX.md；如果没有新的问题，请创建实施文档.md。",
  "contextRelationPrompt": "当前目录中的需求.txt、vX.md、对vX的回复.txt、实施文档.md 构成完整需求沟通过程。请按时间顺序阅读这些文件，并优先重建上下文。"
}
```


### 9.4 `ui.json`

职责：

1. 存放按钮文字
2. 存放提示语
3. 存放错误信息

建议结构：

```json
{
  "buttons": {
    "setPath": "定位切换",
    "createRequirement": "创建需求.txt",
    "copyRequirementPrompt": "复制需求文件提示词",
    "createReply": "创建对vX的回复.txt",
    "copyReplyPrompt": "复制回复文件提示词",
    "copyRelations": "复制文件关系说明"
  },
  "messages": {
    "pathRequired": "请先设置当前主题目录",
    "pathInvalid": "路径不是有效目录",
    "versionFileMissing": "当前目录下未找到 vX.md 文件",
    "fileExists": "文件已存在",
    "copySuccess": "已复制到剪贴板"
  }
}
```


### 9.5 `app.json`

职责：

1. 存放应用级行为配置
2. 存放全局快捷键
3. 存放窗口尺寸
4. 存放窗口初始位置策略

建议结构：

```json
{
  "hotkey": "F2",
  "window": {
    "width": 520,
    "height": 220,
    "startPinnedTop": true,
    "startVisible": true
  },
  "behavior": {
    "closeToTray": true,
    "minimizeToTray": true
  }
}
```

## 11. 模块拆分设计

### 11.1 `configService`

职责：

1. 读取 `prompts.json`
2. 读取 `ui.json`
3. 读取 `app.json`
4. 在程序启动时加载配置
5. 向其他模块提供统一配置读取接口


### 11.2 `fileService`

职责：

1. 校验目录路径是否存在
2. 判断路径是否为目录
3. 创建 `需求.txt`
4. 扫描目录中的 `vX.md`
5. 识别最新版本号
6. 生成对应 `对vX的回复.txt` 路径
7. 创建回复文件


### 11.3 `promptService`

职责：

1. 读取提示词模板
2. 用真实路径替换模板变量
3. 生成需求提示词
4. 生成回复提示词
5. 生成文件关系说明


### 11.4 `clipboardService`

职责：

1. 将生成后的文本写入系统剪贴板


### 11.5 `ideService`

职责：

1. 调用本地 IDEA 打开指定文件
2. 封装打开命令
3. 统一处理打开失败提示


### 11.6 `windowManager`

职责：

1. 创建主窗口
2. 设置窗口置顶
3. 控制窗口显示与隐藏
4. 控制窗口最小化到托盘
5. 控制窗口恢复显示
6. 控制拖拽区域策略


### 11.7 `trayManager`

职责：

1. 创建托盘图标
2. 设置托盘菜单
3. 响应托盘显示命令
4. 响应托盘退出命令


### 11.8 `hotkeyManager`

职责：

1. 注册全局快捷键
2. 监听快捷键触发
3. 在触发时呼出或激活窗口

## 13. 数据流设计

### 13.1 设置目录流程

1. 用户输入路径
2. UI 调用 `fileService.validateDirectory`
3. 校验成功后更新当前目录状态
4. UI 刷新当前目录显示


### 13.2 创建 `需求.txt` 流程

1. 用户点击按钮
2. UI 检查当前目录是否已设置
3. 调用 `fileService.createRequirementFile`
4. 若创建成功，显示结果
5. 若开关开启，调用 `ideService.openFile`


### 13.3 创建回复文件流程

1. 用户点击按钮
2. UI 检查当前目录是否已设置
3. `fileService` 扫描最新 `vX.md`
4. `fileService` 生成 `对vX的回复.txt`
5. 若创建成功，显示结果
6. 若开关开启，调用 `ideService.openFile`


### 13.4 复制需求提示词流程

1. 用户点击按钮
2. UI 检查当前目录是否已设置
3. `promptService` 生成需求提示词
4. `clipboardService` 写入剪贴板
5. UI 提示复制成功


### 13.5 复制回复提示词流程

1. 用户点击按钮
2. UI 检查当前目录是否已设置
3. `fileService` 识别最新版本
4. `promptService` 生成回复提示词
5. `clipboardService` 写入剪贴板
6. UI 提示复制成功


## 14. 错误处理规则

### 14.1 未设置目录

表现：

1. 除“定位切换”外，其余按钮禁用
2. 悬停提示 `请先设置当前主题目录`


### 14.2 路径无效

提示：

1. `路径不存在`
2. 或 `路径不是有效目录`


### 14.3 当前目录无 `vX.md`

影响按钮：

1. `创建对vX的回复.txt`
2. `复制回复文件提示词`

提示：

1. `当前目录下未找到 vX.md 文件`


### 14.4 文件已存在

提示：

1. `文件已存在`

后续行为：

1. 若 IDEA 开关开启，则继续打开已有文件


## 15. IDEA 打开策略

### 15.1 行为规则

1. 只打开具体文件，不打开目录
2. 成功创建文件后，若开关开启，则打开文件
3. 文件已存在时，若开关开启，也打开文件


### 15.2 第一版建议

将 IDEA 调用封装为独立服务，便于后续替换为：

1. VS Code
2. Notepad++
3. 其他编辑器

但第一版默认只服务 IDEA。
